## 1. 概要

本ドキュメントでは、ソリューションセールス伴奏AIにおける**経営アジェンダシート**および**キーパーソンマップ**との連携設計を定義します。

### 1.1 伴奏AIにおけるアカウントプランの位置づけ

```
┌─────────────────────────────────────────────────────────────────────┐
│          アカウントプランと伴奏AIの関係                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【アカウントプラン】                                                │
│  ├─ 経営アジェンダシート: 顧客の経営課題・優先事項の整理            │
│  └─ キーパーソンマップ: 意思決定者・影響者の関係図                  │
│                                                                     │
│  【伴奏AIの役割】                                                    │
│  ├─ 入力として活用: 現状のアカウントプランを理解してニーズ推定      │
│  ├─ 出力として提案: 深掘りポイント・関係深化アクションを提示        │
│  └─ 更新提案: アカウントプランの更新案を出力                        │
│                                                                     │
│  【サイクル】                                                        │
│  アカウントプラン → 伴奏AI → ネクストアクション → 営業活動           │
│        ↑                                           │                 │
│        └───────── 新情報をもとに更新 ←─────────────┘                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. 経営アジェンダシート連携

### 2.1 経営アジェンダシートとは

顧客の経営課題・中期計画・優先事項を整理したドキュメント。営業担当者が面談等で収集した情報をもとに作成・更新する。

### 2.2 入力としての活用

| 活用場面 | 活用方法 |
|----------|----------|
| ニーズ推定 | 経営アジェンダの優先課題からニーズを推定 |
| 提案方針策定 | 中期計画との整合性を確認して提案方針を策定 |
| 情報充足度判定 | アジェンダの記載状況から情報の充足度を判定 |

### 2.3 出力としての提案

#### 経営アジェンダ深掘りアクション

AIは以下の観点から「深掘りすべきポイント」を提示します：

```
【経営アジェンダ深掘りアクション例】

1. 未確認項目の特定
   - 「中期経営計画の投資優先順位が未確認です」
   - 「事業承継の具体的な時期・方針が不明です」

2. 深掘り質問の提案
   - 「投資の優先順位について、次回面談で確認することを推奨します」
   - 質問例: 「中計達成に向けて、投資の優先順位はどうお考えですか」

3. 追加情報源の提案
   - 「IR資料に中計の詳細が記載されている可能性があります」
   - 「業界ニュースで競合の動向を確認することを推奨します」

4. 更新タイミングの示唆
   - 「前回更新から6ヶ月経過しています。最新情報の確認を推奨します」
```

### 2.4 事例の活用

**経営アジェンダ深掘り事例**として、類似企業でのアプローチを参考情報として提示します：

```
【事例】製造業A社での経営アジェンダ深掘り

初回認識: 「コスト削減」が課題
  ↓ 深掘り質問「コスト削減の先に何を目指していますか」
2回目: 「海外展開で競争力強化」が真の目標と判明
  ↓ 深掘り質問「海外展開の最大の障壁は何ですか」
3回目: 「為替リスク」が最大の懸念と特定

→ 「為替リスク管理」というニーズを発見し、為替予約を提案・成約
```

## 3. キーパーソンマップ連携

### 3.1 キーパーソンマップとは

顧客企業の意思決定者・影響者とその関係性を可視化したマップ。誰にどうアプローチすべきかを判断するための情報。

### 3.2 入力としての活用

| 活用場面 | 活用方法 |
|----------|----------|
| 提案アプローチ設計 | 誰にどの順序で提案すべきかを判断 |
| リスク分析 | 決裁プロセスのボトルネックを特定 |
| 関係性の把握 | 当社との関係の深さ・課題を理解 |

### 3.3 出力としての提案

#### キーパーソン関係深化アクション

AIは以下の観点から「関係深化すべきポイント」を提示します：

```
【キーパーソン関係深化アクション例】

1. 接点拡大の提案
   - 「社長との接点が薄いです。以下のアプローチを推奨します」
   - アプローチ例: 「経営者向けセミナーへの招待」

2. キーマン特定の支援
   - 「今回の提案では常務Aが実質的なキーマンと想定されます」
   - 「経理部長は常務への影響力があるため、味方につけることが重要です」

3. 関係深化のステップ提案
   - Step1: 経理部長との関係を活かして常務を紹介してもらう
   - Step2: 常務との面談で投資方針を確認
   - Step3: 常務経由で社長への提案機会を設定

4. 注意点の提示
   - 「社長は保守的な傾向があるため、段階的なアプローチが有効です」
   - 「前任担当者との関係が悪化した経緯があります。慎重に進めてください」
```

### 3.4 事例の活用

**キーパーソン関係構築事例**として、類似企業でのアプローチを参考情報として提示します：

```
【事例】卸売業B社でのキーパーソン関係構築

初期状態: 経理担当者のみと接点
  ↓ 経理担当者から経理部長を紹介
3ヶ月後: 経理部長との関係構築
  ↓ 経理部長から「常務が興味を持っている」と情報入手
6ヶ月後: 常務との面談機会を設定
  ↓ 常務向けの業界動向レポートを提供
9ヶ月後: 常務から社長への橋渡し
  ↓ 社長向けの経営者セミナーに招待
12ヶ月後: 社長との関係構築、大型案件受注

→ 段階的なアプローチで1年かけて関係構築、最終的に大型案件を受注
```

## 4. アカウントプラン更新提案

### 4.1 更新提案の出力形式

```
【アカウントプラン更新提案】

■ 経営アジェンダシート更新案
  追加項目:
    - 「設備投資の優先順位」を追加（確認事項として）
    - 「海外展開の時期」を追加（面談で言及あり）
  変更項目:
    - 「事業承継」の優先度を「中」→「高」に変更
      理由: 社長年齢（65歳）と後継者未定の状況を踏まえ
  削除項目:
    - なし
  情報ソース:
    - 2026年1月15日面談

■ キーパーソンマップ更新案
  追加人物:
    - 経理部長（今回の面談で存在を把握）
      役職: 経理部長
      決裁権限: 経理関連
      当社との関係: 未接点
      備考: 常務への影響力あり（経理担当者情報）
  関係性更新:
    - 常務Aとの関係を「良好」に更新（面談で好感触）
  課題更新:
    - 「社長との接点構築」を課題として明記
```

### 4.2 更新提案のトリガー

| トリガー | 更新提案の内容 |
|----------|----------------|
| 面談実施後 | 面談で得られた新情報の反映提案 |
| 新しい人物の発見 | キーパーソンマップへの追加提案 |
| 優先度変更の示唆 | 経営アジェンダの優先度変更提案 |
| 情報の陳腐化 | 更新されていない項目の確認提案 |

## 5. 事例データベース設計

### 5.1 事例の種類

| 事例タイプ | 内容 | 活用場面 |
|------------|------|----------|
| 商材提案事例 | 類似企業への商材提案（成約/失注） | 提案方針策定 |
| 経営アジェンダ深掘り事例 | 経営課題の深掘りプロセス | 深掘りアクション提示 |
| キーパーソン関係構築事例 | キーパーソンへのアプローチ | 関係深化アクション提示 |
| アカウントプラン運用事例 | 継続的なアカウントプラン更新 | 運用方法の参考 |

### 5.2 事例の収集・蓄積

```
┌─────────────────────────────────────────────────────────────────────┐
│                    事例の収集・蓄積フロー                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【収集タイミング】                                                  │
│  ├─ 案件クローズ時: 成約/失注とその要因を記録                       │
│  ├─ 定期レビュー時: 成功した深掘り・関係構築プロセスを整理          │
│  └─ ナレッジ共有会: 営業知見者のベストプラクティスを収集            │
│                                                                     │
│  【蓄積形式】                                                        │
│  ├─ 構造化データ（YAML/JSON）として蓄積                             │
│  ├─ 類似検索用のメタデータ（業種・規模・状況）を付与                │
│  └─ 匿名化済みの状態で保管                                          │
│                                                                     │
│  【活用方法】                                                        │
│  ├─ システムプロンプトにFew-shot例として投入                        │
│  ├─ 類似企業の事例を参考情報として出力に含める                      │
│  └─ 将来: RAG/Vector DBで類似事例を自動検索                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 6. PoCでの検証ポイント

| 検証項目 | 内容 | 成功基準 |
|----------|------|----------|
| 入力の有効性 | 経営アジェンダ・キーパーソンマップを入力に含めることで推定精度が向上するか | 精度向上が確認できる |
| 深掘りアクションの妥当性 | AIが提示する深掘りポイントは営業知見者から見て妥当か | 営業知見者評価で70%以上が「妥当」 |
| 関係深化アクションの妥当性 | AIが提示する関係深化アクションは実行可能か | 営業知見者評価で70%以上が「実行可能」 |
| 事例活用の効果 | アカウントプラン運用事例を参考情報として提示することで出力品質が向上するか | 品質向上が確認できる |
| 更新提案の有用性 | AIが提示する更新提案は有用か | 営業知見者評価で60%以上が「有用」 |
