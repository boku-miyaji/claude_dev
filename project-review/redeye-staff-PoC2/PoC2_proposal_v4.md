# スタッフタイプ判定AI PoC Phase 2 提案書（v4）

## エグゼクティブサマリ

### Phase 1 の成果と課題

| 項目 | Phase 1 結果 | Phase 2 目標 |
|-----|-------------|-------------|
| 自己認識一致率 | 9割以上 | 70%以上維持 |
| 面談時間（音声） | 7分 | **5〜6分** |
| 面談時間（テキスト） | 10分 | **7〜8分** |
| MT活用検証 | 未実施 | 実施 |

### Phase 2 の目的

**面談時間を短縮しながら、本人一致率を維持し、MT活用可能性を検証する**

## 2. 実施内容

### 2.1 スタッフタイプ判定実現性向上

#### 実施項目

| 項目 | 内容 |
|-----|------|
| **タイプと診断軸の調整** | 15軸→6軸、24タイプ→12タイプ（2軸×4象限モデル） |
| **面談時間削減** | 終了条件緩和、質問優先度最適化、回答時間短縮 |
| **UX改善** | 音声認識精度向上、UI改善、進捗表示 |
| **スタッフ模擬LLM検証** | 想定ペルソナを定義し、自動面談・自動評価 |

#### 評価方法

**定量指標**

| 指標 | 成功基準 | 測定方法 |
|-----|---------|---------|
| **面談時間** | 音声5〜6分、テキスト7〜8分 | 受験ログ分析 |
| **模擬LLMタイプ一致率** | 90%以上 | 自動評価 |
| (参考) 質問数 | 8〜10問 | ログ分析 |
| (参考) 回答時間/質問 | - | ログ分析 |
| (参考) 回答文字数 | - | ログ分析 |

**定性指標（スタッフアンケート）**

| 指標 | 質問 | 成功基準 |
|-----|------|---------|
| **診断結果の納得感** | 診断結果が自分に合っていると思うか | 70%以上が4以上（5段階） |
| **レポートの有用性** | 面談時間と比較して、自己理解に役立つと思うか | 70%以上が4以上 |
| **面談体験** | 面談の体験としてストレスなかったか | 70%以上が4以上 |

## 3. 検証フロー

**ポイント**: スタッフへの実施は**1回のみ**。少数で改善を回し、最終集計を1回で実施。

```
【検証フロー】

Phase A: 少数改善サイクル（Month 1-2）
├── 対象: 社内テスト + スタッフ模擬LLM

│   ├── 模擬LLMでの自動面談・自動評価
│   ├── 少数の社内テスト（5〜10名）
│   ├── パラメータ・UX調整
│   └── 営業ヒアリング・MTレポート改善
└── 成果物: 改善済み面談システム、MTレポートテンプレート

名前: 田中さん（仮）

職種経験: 事務職5年

特徴:
- 仕事の進め方: 自分で判断して素早く動きたい
- コミュニケーション: チームを引っ張るが、メンバーの気持ちも大事にする
- 志向性: 安定した環境でチームと一緒に働きたい

回答傾向:
- 「任せてもらえると嬉しい」

```
