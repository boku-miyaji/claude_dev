# AI開発部署

## 役割
LLM/AIシステムの要件定義・設計・実装・評価・運用を一貫して担当する。
回路設計支援システムのAIコア部分（仕様解析、ブロック図生成、部品選定等）を開発する。
`circuit_diagram/` リポジトリの実コードを読み書きして設計→実装を完遂する。

## チーム構成

| チーム | フォルダ | 役割 |
|--------|---------|------|
| 要件定義 | requirements/ | ユースケース整理、AI適用可否判断、要件整理 |
| 設計 | design/ | アーキテクチャ設計、プロンプト設計、モデル選定 |
| 実装 | implementation/ | LLM API連携、RAG構築、エージェント実装 |
| アルゴ | algorithm/ | アルゴリズム検討、最適化、モジュール化 |
| 評価 | evaluation/ | 精度評価、ベンチマーク、品質保証 |
| AIOps | aiops/ | デプロイ、モニタリング、運用 |

## 標準フロー
要求受領 → 要件定義 → 設計 → 実装 + アルゴ → 評価 → AIOps

必ずしも全チームを経由する必要はない。タスクの規模に応じて柔軟にスキップ可。
小さな修正なら実装チームだけで完結してよい。

## ルール
- 外部（sys-dev等）への提供物は必ずAPI仕様/モジュール仕様を明記する
- 評価チームを経由せず本番投入しない（緊急ホットフィックスを除く）
- プロンプトは設計チームが管理。実装チームが勝手に変更しない
- アルゴチームの成果物は必ずモジュール化し、依頼元が使えるインターフェースで提供
- 各チームのファイルは `チームフォルダ/kebab-case-title.md`
