# 部署連携パイプラインマップ

> 秘書がタスクを受けた時、このマップに従って部署を起動・連携する。

## 並列実行の原則

**依存関係がなければ並列で走らせる。** Claude Code の Agent tool は複数同時呼び出しが可能。

```
並列可能の判定:
  A の出力を B が必要とする → 直列（A → B）
  A と B が互いの出力を必要としない → 並列（A ∥ B）
```

秘書は Agent tool を**1つのメッセージで複数同時に呼ぶ**ことで並列実行する。

## パイプライン一覧

### A: 新機能開発
```
社長「〇〇を作って」

Step 1 [並列]:
  ├── Agent(リサーチ): 技術調査・競合分析
  └── Agent(UXデザイン): ユーザーシナリオ分析
      ↓ 両方完了を待つ

Step 2 [並列]:
  ├── Agent(AI開発): 設計書作成（リサーチ+UXの結果を入力）
  └── Agent(PM): チケット化（UXの依頼事項を入力）
      ↓ AI開発完了を待つ ← checkpoint: 社長レビュー

Step 3 [直列]:
  → Agent(システム開発): 実装（設計書を入力）
  → Agent(QA): テスト
  → 社長に完了報告
```

### B: バグ修正・小さな改善
```
社長「〇〇を直して」

Step 1 [直列]:
  → Agent(システム開発): 調査+修正
  → Agent(QA): テスト確認
  → 社長に完了報告
```

### C: 資料作成
```
社長「資料を作って」

Step 1 [並列]:
  ├── Agent(リサーチ): データ収集
  └── Agent(UXデザイン): 読み手分析（必要に応じて）
      ↓ リサーチ完了を待つ

Step 2 [直列]:
  → Agent(資料制作): Markdown/PPTX作成 ← checkpoint: 社長レビュー
  → 社長に完成報告
```

### D: 調査・分析
```
社長「〇〇を調べて」

Step 1 [並列] — 複数テーマなら分割:
  ├── Agent(リサーチ): テーマA
  └── Agent(リサーチ): テーマB
      ↓ 全完了を待つ

Step 2: 秘書が統合 → notes/に記録 → 社長に報告
```

### E: セキュリティ対応
```
Step 1 [並列]:
  ├── Agent(セキュリティ): 監査・スキャン
  └── Agent(リサーチ): 脆弱性の影響調査
      ↓ セキュリティ完了を待つ

Step 2 [直列]:
  → Agent(システム開発): 修正実装
  → Agent(QA): 修正確認
  → 社長に報告
```

### F: PJ横断タスク
```
社長「全PJの状況を把握したい」

Step 1 [並列] — PJ数だけ同時:
  ├── Agent(リサーチ): foundry の最新状況
  ├── Agent(リサーチ): rikyu の最新状況
  └── Agent(リサーチ): circuit の最新状況
      ↓ 全完了

Step 2: 秘書が統合ダッシュボード → 社長に報告
```

## ハンドオフプロトコル

### 成果物の末尾に書く形式

各部署は成果物の末尾に以下のセクションを書く。秘書がこれを検出して次の部署を起動する:

```markdown
## ハンドオフ

### → PM部への依頼
- [ ] 上記改善案をタスクチケットに分割
- [ ] P0施策の期限設定

### → AI開発部への依頼
- [ ] 〇〇のAPI仕様策定

### → 資料制作部への依頼
- [ ] クライアント向け説明資料の作成
```

### 秘書の検出ロジック

成果物ファイルに `→ {部署名}部への依頼` が含まれていたら:
1. 依頼内容を抽出
2. 該当部署の Agent を起動（Agent tool 経由）
3. 起動ログを activity_log に記録
4. 完了を待ち、次のハンドオフがあればチェーン

## 部署間の依存関係

```
        ┌─────────────┐
        │   情報収集    │ ← 定期実行(cron)
        └──────┬──────┘
               ↓ (ニュース・トレンド)
        ┌──────────────┐
        │   リサーチ     │ ← 調査依頼
        └──────┬───────┘
               ↓ (調査レポート)
    ┌──────────┼──────────┐
    ↓          ↓          ↓
┌────────┐ ┌────────┐ ┌────────┐
│UXデザイン│ │AI開発   │ │資料制作 │
└───┬────┘ └───┬────┘ └────────┘
    │          │
    ↓          ↓
┌────────────────────┐
│     PM（チケット化）  │
└────────┬───────────┘
         ↓
┌────────────────────┐
│  システム開発（実装）  │
└────────┬───────────┘
         ↓
┌────────────────────┐
│     QA（テスト）     │
└────────────────────┘
         ↓
      完了報告
```

## セキュリティ部（横断）

セキュリティ部は全パイプラインに横断的に関与:
- リリース前のセキュリティレビュー
- 依存関係の脆弱性スキャン
- 定期監査（月次）
