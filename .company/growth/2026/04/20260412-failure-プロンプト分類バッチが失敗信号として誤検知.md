# プロンプト分類バッチが失敗信号として誤検知

- **type**: `failure`
- **date**: 2026-04-12
- **category**: process / **severity**: medium
- **status**: active
- **source**: manual
- **tags**: auto-detected, daily-batch, claude-dev

## what_happened
プロンプト分類用のシステムプロンプト（Classify each prompt...）が351件も『correction』シグナルとして記録され、失敗分析バッチ自体もシグナル化する無限ループ状態になっている

## root_cause
失敗信号収集ロジックがバッチ処理用のシステムプロンプト（classify/analyze系）を user correction と誤判定している。自己参照フィルタも無いため、失敗分析バッチの実行自体が次回の入力シグナルになる

## countermeasure
シグナル収集側でバッチ用プロンプト（『Classify each』『Analyze these failure signals』等のプレフィックス、またはバッチ実行者のsession_id/tag）を除外リストに追加し、自己参照を防ぐ。加えて同一文面の重複を dedupe してから分析対象にする
