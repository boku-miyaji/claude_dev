## 📄 注目論文（新着・arXiv 2025〜2026）

### 1. [Agentic Plan Caching: Test-Time Memory for Fast and Cost-Efficient LLM Agents](https://arxiv.org/abs/2506.14852)

**(2506.14852, arXiv, NeurIPS 2025, 提出: 2025-06-17, 最終版: 2026-01-26)**

**背景・目的**: LLM agent アプリケーションにおいて「planning 段階での計算コスト」が顕著。従来の方法は「各タスクごとに scratch からplan を生成」していた。Agentic Plan Caching は「同じ planning パターンを複数タスクで再利用」する仕組みを提案。

**手法**:
- **Plan extraction**: Planning stage から structured plan template を抽出
- **Adaptive reuse**: Task-specific context に基づき過去の plan をカスタマイズ
- **Lightweight models**: Cost-efficient adaptation（full LLM call ではなく軽量モデル利用）

**主要結果**:
- **Cost reduction**: ~50% 削減
- **Latency reduction**: ~27% 削減
- **Performance**: 元の agent performance を維持しながら上記達成

**HD運営への示唆**: focus-you の「morning quote / framework auto-update」も plan caching の対象。「朝のバッチで同じ framework recompute パターン」を cache して cost 削減可能。

---

### 2. [Agentic Code Reasoning](https://arxiv.org/abs/2603.01896)

**(2603.01896, arXiv, 提出: 2026-03-02, v2: 2026-03-04)**

**背景・目的**: LLM agent が「codebase を読んで理解できるか、code を実行せずに」というリサーチクエスチョン。従来の agent は「trial-and-error / execution-based」が多かった。本論文は「structured reasoning（semi-formal approach）」を提案。

**手法**: Semi-formal reasoning
- **Explicit premises**: 前提条件を明示的に列挙
- **Execution trace**: プログラムの実行パスを追跡（実際には実行しない）
- **Formal conclusions**: 導き出した結論を形式化

**評価タスク・結果**:
1. **Patch equivalence verification**: 78% → 88% accuracy
2. **Fault localization (Defects4J)**: Top-5 精度 +5 percentage point
3. **Code question answering (RubberDuckBench)**: 87% accuracy（SOTA）

**focus-you への示唆**: Narrator が「ユーザーの growth arc を読み解く」際に「semi-formal reasoning」パターンを応用可能。Diary entry の implicit intent を explicit にして arc の章立てを提案。

## 💡 focus-you への示唆

### 取り入れるべき

- **Claude Managed Agents Memory の audit log + rollback パターンを Diary edit history に統合**
  → Rakuten の「97% error 削減」は「記憶が正確・トレーサブルであること」の価値を実証。Diary の version control（誰がいつ何を編集）を Memory と同じ design で実装。
  - 対象: `supabase/functions/ai-agent/memoryAgent.ts`, Diary edit API
  - 工数: **中**（audit log 機構の追加）
  - 優先度: **高**（trust & transparency が core feature）

- **Agentic Plan Caching の「template reuse」パターンを morning digest / framework auto-update に適用**
  → Cost 50% 削減実績が確認された。同じ framework recompute パターン（毎朝 7:00 AM）を cache して batch cost 最適化。
  - 対象: `.claude/hooks/batch/`, framework recompute logic
  - 工数: **小〜中**（cache layer の追加）
  - 優先度: **中**（cost efficiency）

- **Semi-formal reasoning を Narrator の「Arc/Theme/Chapter 提案」に活用**
  → Agentic Code Reasoning の「87% accuracy」は structured reasoning の power を実証。Diary の implicit intent → explicit narrative の推論を semi-formal にすることで精度向上可能。
  - 対象: `supabase/functions/ai-agent/narratorAgent.ts`
  - 工数: **大**（prompting と retrieval logic の再設計）
  - 優先度: **中**（ narrative quality）

### 検討に値する

- **Rakuten の「multi-session memory」パターンの参考**
  → Managed Agents Memory が multi-agent concurrent access で 97% error 削減。focus-you を「共有 Diary」に拡張する際の reference architecture として価値あり。
  - 判断ポイント: 将来 focus-you を「ファミリー版」「チーム版」に拡張する予定があるか

### 現状で正しいと確認できたもの

- **Diary を「immutable append-only log」として設計していること**
  → Managed Agents Memory の「audit log」design が focus-you の Diary versioning と conceptually aligned。記憶の integrity を保証する standard approach として学術的に位置づけられた。

- **Morning digest / batch processing を「決まった時間」に実行していること**
  → Plan Caching の「template reuse」効果（cost 50% 削減）で、同じ pattern の繰り返し batch が cost-efficient であることが実証された。

  - 優先度: **中**（next quarter focus）

- **Plan Caching パターンを Hook / Batch の cost 最適化に活用**
  → 50% cost 削減は significant。Hook からの API 呼び出しで「同じ operation template」を cache すれば、Hook cost も削減可能。
  - 対象: `.claude/hooks/api/sb.sh`（キャッシュレイヤー の追加）
  - 工数: **小**（cache layer の軽量実装）
  - 優先度: **低**（コスト最適化は next phase）

### 検討に値する

- **Agentic Code Reasoning の「semi-formal reasoning」パターンを Growth Event 分類に応用**

  - 判断ポイント: 現在の growth-detector.sh が「LLM classification」と「keyword matching」のどちらを主とするか

- **Memory を「externalization の中核」に据えていること**
  → Managed Agents Memory + Plan Caching の 2つのソースで「記憶と計画の最適化」が concurrent に進行していることが確認。focus-you と HD 運営の両面で memory-centric design が正しい方向。
