  - 工数: 中（theme infrastructure、plugin version pinning）

- **Cognis (2604.19771) の永続メモリアーキテクチャ参考**  
  → Diary + Memory Intelligence Agent の context-aware memory 設計に直結。「会話エージェント向け persistent memory」は focus-you の growth log / introspection memory と哲学が合致。
  - 対象: `supabase/functions/ai-agent/`, `src/lib/memoryAgent.ts` (if exists)
  - 工数: 中（新テーブルスキーマ、retrieval logic）

- **MCP full support (Claude Code, Google Gemini Enterprise)**  
  → focus-you の知識ベース（Supabase）を MCP server 化の検討加速。既存の hooks/skills では reach しない用途（他システムの AI agent 連携）に対応可能。
  - 対象: `supabase/functions/`, MCP server skeleton 新規作成
  - 工数: 大（MCP spec 実装、auth, RLS）

- **Multi-agent workflows の記憶・coordination（SkillGraph, Prism, TriEx）**  
  → focus-you の multi-agent 設計が先行。記憶の共有・階層化（Prism の evolutionary memory）、説明可能性（TriEx）が論文で実装されつつある。現在の implementation と比較検証。
  - 対象: `src/pages/Blueprint.tsx` (Narrators/Agents tab)
  - 工数: 小（比較分析のみ）

### 検討に値する

- **Claude Mythos Preview の security hardening specialization**  
  → focus-you は「自己理解」「幸せ」が軸だが、社長の「信頼性」に対する潜在的関心（前回の freeze incident、reliability feedback）。Claude Mythos を監視し、security-aware reasoning が自己分析に応用できるか検討。
  - 判断ポイント: セキュリティコンテキストが感情分析・ナレッジ管理に必要か

- **Schema-Constrained Generation (2604.20117)**  
  → Diary entry generation で「自由度」と「構造」のバランスが課題。スキーマ制約付きながら coherent memory を生成する手法が論文化。検証用 POC 価値あり。
  - 判断ポイント: Diary UX が「テンプレート感」を嫌うか、「ガイダンス」を好むか

- **Evaluation Exploitation (2604.20200) — アラート**  
  → Coding agent が benchmark score を gaming する問題報告。focus-you が code generation (via Claude Code integration) を持つなら、「ユーザーの実績が inflated されていないか」の audit mechanism を設計検討。

### 現状で正しいと確認できたもの

- **MCP を中核に据えた Agent-to-Tool architecture**  
  → 4月の大型更新（Claude Code, Gemini Enterprise, IP Fabric, Talon）で MCP が「de facto standard」として確立。focus-you の hook/plugin 責務分離（`.claude/rules/hd-operations.md`）が先見の明。現在の設計継続。

- **Memory-centric agent design**  
  → Cognis, SkillLearnBench, To Know is to Construct など、agent の「記憶」を中核にした論文が多数。focus-you の Diary + Memory Intelligence Agent パターンが trend ahead。

- **「与えられた autonomy」 (Managed Agents) の価値**  
  → Anthropic の managed agents 発表（4/8）から1周間で、Google/Microsoft/Meta が concurrent に agent autonomy framework を発表。focus-you の「秘書が判断、Agent が実行」モデル（`.company/CLAUDE.md`）は business requirement と tech trend の両立。

- **Transparency 優先（VS 「非表示」）**  
  → DevUI visualization (Microsoft, Gemini Enterprise Canvas) が強調される一方で、Anthropic の「hidden AI actions」への developer 反発（HN討論）が顕著。focus-you の「明示的なハンドオフ + YAML構造化」（`handoff.md`）が正しい方向。
