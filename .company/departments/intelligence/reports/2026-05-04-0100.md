## 📄 注目論文

### Agent Architecture & Execution Scheduling

- **(2026-04, arXiv:2603.01896) [Agentic Code Reasoning](https://arxiv.org/abs/2603.01896)**
  - **目的**: execution なしでコード理解を可能にする
  - **手法**: semi-formal reasoning (explicit premises, execution traces, formal conclusions) を LLM agent に要求
  - **結果**: 
    - patch equivalence verification: 78% → 88% (curated) / 93% (real-world agent patches)
    - code QA: 87% on RubberDuck benchmark
    - fault localization: +5 pts on Defects4J
  - **技術価値**: RL pipeline, code review, static analysis の executable base化
  - **HD-ops 示唆**: Managed Agent内での code reasoning を semi-formal prompting で実装すれば、検証コストをLLM推論に移譲可

- **(2026-04, arXiv:2604.11378) [From Agent Loops to Structured Graphs: A Scheduler-Theoretic Framework](https://arxiv.org/abs/2604.11378)**
  - **目的**: LLM agent execution の制御可能性・検証可能性を復活させる
  - **提案**: SGH (Structured Graph Harness)
    - implicit control flow (Agent Loop) → explicit DAG
    - 3層planning architecture (plan/execute/recover)
    - 全execution が immutable log として記録可
  - **効果**: token consumption 37% 削減（plan layer に遡る overhead 除去）
  - **ポジション**: expressiveness と controllability のトレードオフ（古典的scheduling theory の適用）
  - **HD-ops への直結示唆**: 
    - HD-ops の `pipeline.md` DAG化 = SGH と同思想
    - current Agent Loop (prompt → tool → loop) を structured graph に移行すれば、token cost低下 + full auditability 実現
    - 部署委譲時の implicit context sharing → explicit handoff YAML ブロック化も同じ方向

### Multi-Agent Coordination & Memory

- **(2026-04, arXiv:2604.19540) [Mesh Memory Protocol: Semantic Infrastructure for Multi-Agent LLM Systems](https://arxiv.org/abs/2604.19540)**
  - **目的**: 複数 agent 間の cognitive state 共有・管理フレームワーク
  - **手法**: semantic memory layer (knowledge graph 相当) を agent 間でmesh 状に接続
  - **実績**: Anthropic Claude Plugin Directory approval (2026-04-18)
  - **focus-you 示唆**: Diary + Memory Agent + Growth Engine が複数 context で動くとき、shared memory pool が必須。今は `secretary_notes` table が実質これ。Mesh protocol を参考に structured できそう

- **(2026-04, arXiv:2604.04853) [MemMachine: A Ground-Truth-Preserving Memory System](https://arxiv.org/abs/2604.04853)**
  - **目的**: agent memory の accuracy 維持（lossy LLM extraction 廃止）
  - **手法**: 
    - 3層: short-term / long-term episodic / profile memory
    - full conversation episodes を保存（summary ではなく raw）
    - contextualized retrieval: nucleus match 拡張で周辺context 取得
  - **結果**: 80% fewer input tokens vs Mem0, 93% benchmark accuracy
  - **実装**: Retrieval Agent が query complexity に応じて戦略選択 (direct / decomposition / chain-of-query)
  - **focus-you 直結示唆**: 
    - Diary import/export で「raw episode」保存が focus-you philosophy ("portability")
    - Memory retrieval の3層化 (working / episodic / semantic) を今の `diary_entries` + `memory_vectors` で実装可
    - LLM extraction を廃止して raw + structured embedding だけに = accuracy向上 + token削減

- **(2026-04, arXiv:2604.16548) [A Survey on the Security of Long-Term Memory in LLM Agents: Toward Mnemonic Sovereignty](https://arxiv.org/abs/2604.16548)**
  - **目的**: agent memory security の包括サーベイ。ガバナンス枠組み定義
  - **主要発見**: 
    - Write/Retrieve 段階の integrity study は充実も、Confidentiality/Availability/Forget は未研究
    - 9種のgovernance primitives が必要 (write authority, access control, modification timing, erasure, etc)
    - Mnemonic Sovereignty = verifiable control over 4 dimensions (what / who / when / which-prior-states)
  - **6phase lifecycle**: Write → Store → Retrieve → Execute → Share → Forget/Rollback
  - **評価軸**: integrity / confidentiality / availability / governance across all phases
  - **focus-you 戦略示唆**: 
    - Diary が個人資産 = mnemonic sovereignty 最高度要求
    - data deletion / GDPR / right-to-be-forgotten = Forget phase
    - 今は emoji reaction や bookmark は logs に残すが、user request で完全削除可能？ → Forget phase governance の実装必須
    - Memory poison (false memory injection) 対策 = raw episode保存 + LLM extraction廃止で natural defense

- **(2026-01, arXiv:2601.01885) [Agentic Memory: Learning Unified Long-Term and Short-Term Memory Management](https://arxiv.org/abs/2601.01885)**
  - **提案**: AgeMem = unified memory management policy (what to store / when to retrieve / what to discard)
  - **仕組み**: memory operations as tool-based actions (store, retrieve, update, summarize, discard を LLM が主導的に呼び出し)
  - **効果**: long-horizon reasoning tasks の token cost 削減 + accuracy向上
  - **HD-ops 示唆**: agent のmemory access を Tool として explicit化 = Structured Graph Harness と同じ方向。Hook で自動記録 vs LLM主導access のバランス
