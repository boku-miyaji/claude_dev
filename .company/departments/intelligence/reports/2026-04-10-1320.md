## 2. Anthropic 公式エンジニアリング

### (2026-04-08) Claude Managed Agents 公開ベータ提供開始

Anthropic が **完全マネージド型エージェント実行インフラ** を公開ベータでリリース。従来は企業が自分でハーネス・サンドボックスを構築する必要があったが、Anthropic が提供。

**アーキテクチャ:**

```

     ↓
Harness      (loop: calls Claude, routes tool calls)
     ↓
Sandbox      (execution environment for code, files)
```

**HD適用**: 当社の部署エージェント（hub-and-spoke モデル）も同様の 3層構造。Anthropic が共通パターンを infrastructure として提供することで、業界標準化が加速。当社ノウハウの 汎用性・再利用性が高まった証。

**情報源**: [Anthropic Engineering: Scaling Managed Agents](https://www.anthropic.com/engineering/managed-agents)

### (2026-04-05) Agent Skills: 実世界への適用フレームワーク

Agent Skills = **specialized agents build via organized folders**

```
my-skill/
├── SKILL.md           (metadata & description)
├── instructions/      (step-by-step guides)
├── scripts/           (runnable utilities)
└── resources/         (static files, templates)
```

**特徴:**

1. **動的発見と読み込み** — エージェントが実行時に skill をロードして特化

**HD適用**: 当社の `.claude/skills/` ディレクトリ構造が業界標準に沿っている。今後は「社内スキル化」→「外部配布」への道筋が明確。

## 3. Claude Code 公式ベストプラクティス

### (2026-04-09) Best Practices for Claude Code — 公式ドキュメント

**コンテキスト管理:**
- ウィンドウ容量を **60%以下** に保つ
- `/clear` で無関連タスク間の分離
- `/compact` で長いセッションの要約

**自律ワークフロー:**
- Claude に「何」と「なぜ」を指示し、「どう」は任せる
- ユーザーは architecture 判断・review・orchestration に注力

**実行前計画:**
- 「このタスクの step-by-step approach を提案して」とリクエスト
- Plan の approval → Step 実行 → 検証 のサイクル

**セキュリティ:**
- Security-critical code は必ず validate
- Routine work を Claude に任せ、creative problem-solving に集中

**PR 管理:**
- 1 feature = 1 PR（レビュー容易・revert 容易）
- 1時間ごと commit 推奨（granular history）

**情報源**: [code.claude.com/docs/en/best-practices](https://code.claude.com/docs/en/best-practices)

### (2026-04-07) 「Production で MCP, Subagents, CLAUDE.md」— 架構レッスン

**プロダクション architecture:**

```
CLAUDE.md          (policies, tool access, rules)
    ↓
MCP Servers        (external integrations: DB, API, UI tools)
    ↓
Subagents          (isolated task workers)
    ↓
Worktrees          (git-based session isolation)
```

**実装ガイドライン:**
1. CLAUDE.md が session governance の source of truth
2. MCP = external world との contract
3. Subagents = fault isolation & parallel work
4. Worktrees = repo state isolation (git worktree via claude --worktree)

**情報源**: [Dev.to: Production Guide](https://dev.to/lizechengnet/how-to-structure-claude-code-for-production-mcp-servers-subagents-and-claudemd-2026-guide-4gjn)

## 5. Medium 高度な記事

### (2026-04-09) Subagents — Complete Guide to Delegation

**Delegation パターン:**

1. **Explore-Plan-Execute** (complex tasks)
   ```
   Subagent-1: Codebase exploration
   Subagent-2: Architecture planning
   Subagent-3: Implementation
   Main: Orchestration & review
   ```

2. **Permission Hygiene**
   - Subagent は最小権限原則で run
   - Main agent が critical approvals

**Subagent vs Agent Teams:**

| Subagent | Agent Teams |
|----------|-------------|
| Single session within isolation | Multiple independent instances |
| Results flow back to main | Shared task list + direct messaging |
| Sequential or limited parallelization | True parallelization |

### (2026-04-06) Agent Teams — True Parallelization

**Agent Teams pattern:**

```
Team Coordinator (task queue, priority)
     ↓
Agent-A (role: backend)
Agent-B (role: frontend)
Agent-C (role: devops)
     ↓
Shared results sink
```

**Subagent との違い:**
- Subagent: main session 内の isolated worker
- Team: 完全独立した複数 Claude instances（本当の parallelization）
