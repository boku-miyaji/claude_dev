### (2026-03-24) Anthropic公式: 長時間開発向けハーネス設計

Anthropic の Engineering Blog がマルチエージェントハーネスの設計哲学を公開。主要な知見:

- **3エージェント構成**: Planner（計画立案）、Generator（コード生成）、Evaluator（品質評価）
- **GAN的ループ**: Generator と Evaluator が敵対的に反復し、品質を収束させる
- **コンテキストリセット**: 長時間セッションで coherence を維持するため、構造化されたハンドオフでコンテキストをリフレッシュ
- **キャリブレーション済み Evaluator**: モデルは自作コードを過大評価する傾向があるため、評価基準を明示的にキャリブレーション
- **実績**: 5-15回の反復、最大4時間の連続セッション。ソロ実行（20分/$9）vs ハーネス実行（6時間/$200）で完成度に大差

## 3. Hooks の新機能・新イベントタイプ

### (2026-04時点) Hook イベント数: 14 -> 21 に拡大

Claude Code Hooks は2026年に大幅拡張され、21のライフサイクルイベント、4つのハンドラータイプを持つ自動化プラットフォームに進化。

**新規追加イベント（2026年）:**
- `InstructionsLoaded` -- CLAUDE.md 等のインストラクション読み込み完了時
- `ConfigChange` -- 設定変更検出時
- `WorktreeCreate` / `WorktreeRemove` -- Git worktree の作成/削除時
- `PostCompact` -- Context Compaction 実行後
- `Elicitation` / `ElicitationResult` -- ユーザーへの質問と回答
- `PermissionDenied` -- auto mode 分類器の拒否時（v2.1.89 追加）

**新規ハンドラータイプ:**
- `HTTP` hook -- イベント JSON を HTTP POST で外部送信
- `Agent` hook -- サブエージェントを spawn してツールを使った深い検証を実行

**アーキテクチャ:**
- リードセッション（1つ）がタスク割り当て・結果統合

**コスト・制限:**
- 単一セッションの3-7倍のトークンコスト
- セッション再開機能なし
- 最適用途: マルチファイルリファクタ、クロスレイヤー機能開発、リサーチ&レビュー

## 5. CLAUDE.md 設計に関する新知見

### (2026-04時点) CLAUDE.md のサイズ制限と Progressive Disclosure

最新のベストプラクティスでは CLAUDE.md の肥大化を明確に警告。

**サイズ目安:**
- 1ファイルあたり 200行以下を目標
- HumanLayer 社は 60行以下で運用
- フロンティアモデルは約150-200個の指示に従える。Claude Code のシステムプロンプトで約50を消費 → 残り100-150が CLAUDE.md の予算

**Progressive Disclosure パターン:**
- CLAUDE.md には「何を知るべきか」ではなく「どこで情報を見つけるか」を書く
- 詳細手順は `references/` 等に分離し、必要時にのみ Read
- これにより instruction count を節約しつつ、必要な深さの情報にアクセス可能

**定期メンテナンス:**
- 数週間ごとに Claude 自体に CLAUDE.md のレビューを依頼
- 冗長な指示の統合、陳腐化した指示の削除、矛盾の解消

**Hook との役割分担:**
- CLAUDE.md = 助言（遵守率 ~80%）
- Hook = 強制（遵守率 100%）
- 必須ルールは Hook に移行すべき

- [Claude Code Best Practices](https://code.claude.com/docs/en/best-practices)
- [UX Planet: CLAUDE.md Best Practices](https://uxplanet.org/claude-md-best-practices-1ef4f861ce7c)
- [HumanLayer: Writing a Good CLAUDE.md](https://www.humanlayer.dev/blog/writing-a-good-claude-md)
- [builder.io: CLAUDE.md Guide](https://www.builder.io/blog/claude-md-guide)

**HD適用**: 当社の `.company/CLAUDE.md` は現在約200行。`references/` への分離は既に実施済みだが、instruction count の観点から更なる簡素化を検討すべき。「Hook で保証、CLAUDE.md で助言」の原則に沿って、必須ルールの Hook 化を推進。
