#### (2026-04-14) Claude パフォーマンス低下の報告相次ぐ。トークン節約のため「努力レベル」を秘密裏に引き下げ
URL: https://fortune.com/2026/04/14/anthropic-claude-performance-decline-user-complaints-backlash-lack-of-transparency-accusations-compute-crunch/

Anthropic がトークン消費削減のため、Claude のデフォルト「effort level」（努力度）を秘密裏に低下させたことが判明。ユーザーから以下の苦情が相次いでいる：

**報告内容**:
- 複雑なエンジニアリングタスクでは「使い物にならない」（AMD シニアディレクター）
- 指示の遵守が悪化。単純化を優先し、文脈を読むための複数ファイル参照を放棄
- 「研究ファースト」から「編集ファースト」へのスタイル変化。エラーが増加、ユーザー介入が必要に

**Microsoft Research Manager の発言**: 「effort を max に設定しても、極度に雑で、指示を無視し、同じ誤りを繰り返す」

**背景**: Anthropic は「ユーザーフィードバックから過度なトークン消費があった」と主張。しかし透明性の欠落と、最近のコンピューティング容量逼迫（障害やピーク時の使用上限厳格化）から、インフラ不足が根本原因の可能性が指摘されている。

#### (2026-04-08) Anthropic Managed Agents が本格展開。セッション永続化と複数エージェント オーケストレーションが標準化
URL: https://dev.to/bean_bean/claude-managed-agents-deep-dive-anthropics-new-ai-agent-infrastructure-2026-3286

Anthropic が 2026-04-08 に **Claude Managed Agents** をパブリックベータで発表。Cloud ホストの AI エージェント開発プラットフォーム。

**コア機能**:
1. **セッション永続化**: メモリストアがセッション間で永続。ネットワーク切断も引き継ぎ。
2. **サンドボックス**: 本番インフラ上で安全に実行。認証・自動状態永続化を備える。
3. **マルチエージェント オーケストレーション（研究プレビュー）**: オーケストレーター Claude が複数サブエージェントをスポーン・並列委譲・結果を集約。
4. **MCP 統合**: 標準プロトコルでツール・外部システムへのアクセス。
5. **実行トレース**: 完全な監査ログ。

**アーキテクチャの設計思想**: 「Brain（Claude）」と「Hands（サンドボックス）」の分離。脳は状態操作と計算を指示、手はそれを実行。スケーリングはブレイン×ハンズの独立スケールで対応。

## 💡 focus-you への示唆

### 取り入れるべき

- **Claude Code Routines** → HD 運営改善部のバッチ・定期タスク自動化へ
  - 対象: `.claude/hooks/`, `.company/secretary/`
  - 工数: 小
  - HD 秘書が定期的に実行する「毎日のプロンプト解析」「weekly digest 作成」などは、Routines として Cloud にスケジュール登録可能。Mac のオンライン状態に依存しない信頼性向上。

- **Managed Agents のセッション永続化・マルチエージェント オーケストレーション** → focus-you のパイプライン実装フェーズで標準採用
  - 対象: `company-dashboard/supabase/functions/ai-agent/`, `.company/departments/*/` の部署エージェント設計
  - 工数: 中
  - 現在 focus-you は「社長指示 → 複数部署並列実行 → ハンドオフ」という DAG を自前実装している。Managed Agents の Orchestrator パターンを採用すれば、セッション永続化・状態共有・エラー再試行がプラットフォーム提供になり、HD 運営ロジックの簡素化が可能。特にパイプラインが中断→復帰する際のコンテキスト保全が重要。

- **MemAgent・LoCoMo の長期メモリ研究** → Narrator の感情 Arc 追跡に適用
  - 対象: `company-dashboard/src/lib/aiPartner.ts`, `emotion_analysis` テーブル, knowledge_base
  - 工数: 中
  - ユーザーの「自己理解の旅」は数ヶ月～数年単位。MemAgent の RL ベース記憶圧縮と LoCoMo の長期一貫性評価を組み合わせれば、「6ヶ月前のこの気づきが今につながっている」という物語性を自動的に抽出可能。感情ベクトルの時系列追跡と相互参照。

- **日本語 LLM の活用（ELYZA等）** → focus-you 国内展開時の品質向上
  - 対象: Claude + OpenAI API とのハイブリッド
  - 工数: 小～中
  - 現在 Claude 一本槍だが、日本語の微細なニュアンス（敬語・感情表現・文脈依存性）はELYZA等の日本語特化モデルが優位。感情分析やナラティブ生成で Claude と ELYZA を使い分ける hybrid アーキテクチャ。

### 検討に値する

- **Meta の原則ベース安全性** → CLAUDE.md / rules/ の「なぜ」の層厚化
  - Meta が「ルールベースではなく『なぜそのルールが存在するか』を LLM に理解させる」方針を強化。focus-you の Agent（部署エージェント）が単に CLAUDE.md を遵守するのではなく、その背後の「設計思想」を理解して判断できるように。ナレッジ蓄積・昇格ルールの説明責任を強化。

- **Claude パフォーマンス低下への警戒** → API コスト増への対策検討
  - Anthropic が意図的に effort を下げているなら、Claude API の「品質 / コスト」が今後悪化する可能性。GPT-5.4 や Gemma 4（オープン化）への部分移行、Managed Agents での処理の最適化（思考圧縮など）を検討。

- **日本の「物理 AI」トリリオンモデル計画** → ロボティクス・スマートホーム連携の素地
  - 2030年の物理 AI 実装が実現すれば、focus-you の「自己理解」が「家・ロボット・デバイスとの相互作用」へ拡張される可能性。現時点では直接的なビジネス機会ではないが、長期的な市場動向として注視。

### 現状で正しいと確認できたもの

- **Anthropic Managed Agents の設計思想（Brain/Hands 分離）** → HD パイプライン設計（Hook/company 分離）と一致
  - Brain が状態を読み、計画を立て、Hands が実行する。hooks が記録し、/company が判断・行動する設計は正しい。Anthropic がインダストリアル・スタンダードとしてこれを推奨している確認。

- **セッション永続化 = エージェント システムの必須要件**
  - LoCoMo や Managed Agents での論文・実装の標準化により、「長期メモリなきエージェントは不完全」という合意が成立。focus-you の session_id ベース記録設計は先見性あり。

- **多言語 AI エコシステム化（日本語 LLM, Gemma 等オープン化）** → ベストツール選択を正当化
  - 「Claude 一本」ではなく「Claude + GPT + 日本語モデル」のハイブリッドが業界スタンダードへ。focus-you の「用途別 LLM 選択」方針は正しい判断。
