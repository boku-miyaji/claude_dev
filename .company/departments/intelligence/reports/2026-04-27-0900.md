## 2. AI Agent / LLM 業界

### [HyperAgents — Meta の Self-Improving Agent ブレークスルー](https://arxiv.org/abs/2603.19461)

**(2603.19461, arXiv, 発表: 2026-03-19)**

**背景**: 従来の agent は「与えられたプロンプト・ツール・アーキテクチャで固定」だった。Meta / UBC / Oxford / NYU のチームが「Agent が自分の improvement procedure を自分で書き換える」メタレベルの自己改善を実装。

**主要成果**:
- **数学オリンピアド問題**: 50問中10問（20%）で既知の最適解を更新。Strassen の行列積アルゴリズム（1969年）を凌駕する新アルゴリズムを発見
- **転移学習**: ロボティクスで学んだ self-improvement パターンを論文査読・数学オリンピアドに転移成功。汎用化の実証
- **経済効果**: Google で実験した automated algorithm optimization で worldwide compute の 0.7% 削減（年数百万ドルコスト削減）

**So-What**: focus-you の Narrator が「ユーザーの自己理解を助ける」際、Agent自身が「どのフレームワーク・プロンプト・カテゴリ分けを使うべきか」を試行錯誤・改善する機能を検討する余地あり。現在は「固定の感情分析ロジック」だが、ユーザー個別に最適なロジックへの自動調整が可能。実装コスト：中。

### [Memory for Autonomous LLM Agents: Mechanisms, Evaluation, and Emerging Frontiers（サーベイ論文）](https://arxiv.org/abs/2603.07670)

**(2603.07670, arXiv, 発表: 2026-03 初旬)**

**内容**: 2022年以降の「LLM Agent における記憶機構」の最新サーベイ。

**Key framework**: 記憶を「write → manage → read」の三層ループと、「temporal scope（短期/長期）」「representational substrate（埋め込み/テキスト/グラフ）」「control policy（自動/手動）」の3軸で分類。

**重要な知見**:
- Agent の capability の中核は記憶。記憶なし agent は「1ターン相談」に限定
- Focus-you でいう「Diary」は agent の long-term memory 化する戦略。記憶がなければ「毎回新しい人格になる」
- 最新論文の傾向：「implicit（埋め込みベクトル）」から「explicit（テキスト・グラフ構造化）」への転換。LLM が読んで理解できる形式が優位

**So-What**: focus-you の Memory Intelligence Agent 設計が論文のトレンドと同期していることを確認。特に「Diary を structured graph（人間関係マップ、成長アーク、反復テーマ）に変換して Agent が参照」というアプローチは、2026年の最新研究と方向性が一致。追加投資する価値あり。

## 3. Supabase / インフラ

### [Supabase API Key 移行（ES256 JWT + sb_publishable_* 形式）](https://github.com/orgs/supabase/discussions/41834)

**04/22 deprecated** — Supabase が API Key 形式を大幅変更：

**変更内容**:
| 項目 | 旧形式 | 新形式 |
|------|--------|--------|
| 公開鍵 | `anon key` | `sb_publishable_...` |
| 秘密鍵 | `service_role key` | `sb_secret_...` |
| JWT署名 | HS256（対称鍵） | ES256（非対称鍵） |

**破壊的変更**:
- Edge Function で **JWT が `Invalid JWT` エラー** で返ることがある。ゲートウェイが ES256 署名を検証できず拒否
- 既存の `anon key` 直接埋め込みコードが 04/22 以降エラーになる可能性
- `.claude/hooks/supabase.env` の `SUPABASE_ANON_KEY` が無効化された可能性

**対応方法**:
- Edge Function は `verify_jwt=false` + 関数内で `sb.auth.getUser(jwt)` で検証（claude-dev の既存 google-calendar-proxy / ai-agent は既に対応済み）
- curl / sb.sh アクセスは `sb_publishable_*` / `sb_secret_*` に置き換え
- 環境変数ファイルを更新：`~/.claude/hooks/supabase.env`

**So-What（緊急）**: claude-dev の全Edge Function（`supabase/functions/`配下）を確認。対応未実施の function があれば修正が必須。次のバッチ実行時に Supabase アクセスが失敗する可能性。確認スコープ：`supabase/functions/ai-agent/`, `google-calendar-proxy/`, 任意の custom function。
