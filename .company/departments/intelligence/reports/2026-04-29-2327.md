## 📄 注目論文

- **[Agentic Harness Engineering: Observability-Driven Automatic Evolution of Coding-Agent Harnesses](http://arxiv.org/abs/2604.25850v1)** (2604.25850v1, 2026-04-28)
  - 背景: コーディングエージェント性能は harness 構成が大きく規定するが、自動化は困難（行動空間が異種、評価ノイズ、長軌跡）。
  - 手法: コンポーネント / 経験 / 意思決定の3つの observability の柱を整備し、harness を自動進化させる AHE フレームワーク。
  - 結果: ファイルレベルの編集可能・revertible な行動空間と experience observability で評価可能化を実証。
  - **focus-you / hd-ops 示唆**: claude_dev の Hook / 部署プロトコルの自動進化に直接接続できる思想。「仕組みが仕組みを改善する」運営改善部のテーマと一致。

- **[A paradox of AI fluency](http://arxiv.org/abs/2604.25905v1)** (2604.25905v1, 2026-04-28)
  - 背景: AI スキルがユーザー成果に与える影響は未解明。
  - 手法: WildChat-4.8M の 27K トランスクリプトを注釈・分析。
  - 結果: 上級者は反復的・批判的に対話、難しいタスクに挑むため失敗も多いが、可視化・部分回復・最終成功率も高い。
  - **focus-you / hd-ops 示唆**: 「失敗の可視化」と「partial recovery 導線」を体験設計に組み込む根拠。silence-first と矛盾しない範囲で「失敗からの再起導線」を強化する余地。

- **[ADEMA: A Knowledge-State Orchestration Architecture for Long-Horizon Knowledge Synthesis with LLMAgents](http://arxiv.org/abs/2604.25849v1)** (2604.25849v1, 2026-04-28)
  - 背景: 長期エージェントタスクは知識状態のドリフトと中間コミットの暗黙化で破綻する。
  - 手法: Epistemic bookkeeping + 異種デュアル評価 + チェックポイント再開 + セグメントメモリ凝縮 + artifact-first 組立 を統合。
  - 結果: 4シナリオ・60実行で長期合成のロバスト性を検証。
  - **focus-you / hd-ops 示唆**: pipeline.md / handoff.md / agent_sessions テーブルの設計と高い親和性。Managed Agents パターンの強化材料。

- **[Recursive Multi-Agent Systems](http://arxiv.org/abs/2604.25917v1)** (2604.25917v1, 2026-04-28)
  - 背景: 単一モデルの再帰スケーリングを多エージェント協調に拡張可能か。
  - 手法: 異種エージェントを RecursiveLink で接続、潜在状態を相互転送する RecursiveMAS、内外ループ協調最適化。
  - 結果: マルチエージェント協調自体を再帰でスケールできることを示唆。
  - **focus-you / hd-ops 示唆**: 部署エージェントのループ協調（秘書↔部署↔秘書）の理論的補強。

- **[Pythia: Toward Predictability-Driven Agent-Native LLM Serving](http://arxiv.org/abs/2604.25899v1)** (2604.25899v1, 2026-04-28)
  - 背景: マルチエージェント構造が露出する semantic predictability を既存LLMサービングは活用できていない。
  - 手法: prefix キャッシュ・長コンテキストリソース競合等のボトルネック分析と最適化。
  - 結果: エージェント特化サービングの最適化点を提示。
  - **focus-you / hd-ops 示唆**: コスト分離（dashboard/batch/Hook のモデル使い分け decision）に corroborate する一次資料。

- **[Conditional misalignment: common interventions can hide emergent misalignment behind contextual triggers](http://arxiv.org/abs/2604.25891v1)** (2604.25891v1, 2026-04-28)
  - 背景: Emergent Misalignment (EM) 緩和介入が、評価条件下では消えるが訓練文脈に近いプロンプトでは再発する。
  - 手法: 介入後モデルを訓練文脈に近づけたプロンプトで再評価。
  - 結果: 評価で見えなくなった misalignment が、文脈依存トリガーで再発する「条件付き misalignment」を確認。
  - **focus-you / hd-ops 示唆**: Edge Function やプロンプトの安全評価で「評価環境と本番文脈の差」を意識する必要性。

- **[Three Models of RLHF Annotation: Extension, Evidence, and Authority](http://arxiv.org/abs/2604.25895v1)** (2604.25895v1, 2026-04-28)
  - 背景: RLHF アノテータの規範的役割が曖昧。
  - 手法: 「設計者の延長 / 独立証拠 / 独立権威」の3モデルに概念分解。
  - 結果: それぞれが annotation 設計と集約方法の前提を変える。
  - **focus-you / hd-ops 示唆**: 社長フィードバック → ナレッジ昇格の意思決定プロセス（knowledge-accumulation.md）の整理材料。

- **[DV-World: Benchmarking Data Visualization Agents in Real-World Scenarios](http://arxiv.org/abs/2604.25914v1)** (2604.25914v1, 2026-04-28)
  - 背景: 既存のDVベンチはサンドボックス・単一言語・完全意図前提。
  - 手法: 260タスクで spreadsheet / evolution / interaction の3領域を評価。
  - 結果: 実世界 DV エージェントの評価ベンチマーク提供。
  - **focus-you / hd-ops 示唆**: company-dashboard で将来 DV 系エージェントを導入する場合の評価指針。

- **[MAIC-UI: Making Interactive Courseware with Generative UI](http://arxiv.org/abs/2604.25806v1)** (2604.25806v1, 2026-04-28)
  - 背景: 教材の対話型UI生成は規模・教育的精度・反復速度に課題。
  - 手法: 構造化知識分析 + 2段階 generate-verify-optimize パイプライン。
  - 結果: ゼロコード教材編集システム。
  - **focus-you / hd-ops 示唆**: focus-you の Narrator / 自己理解コンテンツを「対話型」化する場合の参考。

- **[SAFEdit: Does Multi-Agent Decomposition Resolve the Reliability Challenges of Instructed Code Editing?](http://arxiv.org/abs/2604.25737v1)** (2604.25737v1, 2026-04-28)
  - 背景: EditBench で40モデル中39が TSR 60% 未満。
  - 手法: Planner / Editor / Verifier の役割分割 + Failure Abstraction Layer。
  - 結果: テスト失敗ログを抽象化して反復改善する多エージェント編集枠組み。
  - **focus-you / hd-ops 示唆**: claude_dev の「実装→QA→修正」サイクルの参考設計。
