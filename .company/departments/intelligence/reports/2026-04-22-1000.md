## 📄 注目論文（arXiv April 2026）

### **(2026-04-13) From Agent Loops to Structured Graphs: LLM Agent 実行のスケジューラ理論**

**arXiv**: https://arxiv.org/abs/2604.11378 (v1: 2026-04-13)

**著者**: 論文は LLM agent execution system を scheduler として分析

**要点**:
- **問題設定**: 「Agent Loop」（単一 LLM が無限反復）は implicit dependencies / unbounded recovery loops / mutable history を引き起こし、debugging/verification 困難
- **提案**: **Structured Graph Harness (SGH)** — execution plan を explicit static DAG に lift
- **特徴**:
  - Planning と Recovery を分離（layered escalation protocol）
  - Execution history を immutable に
  - 70 個の既存 agent system を分析したスケジューラー理論を適用
- **trade-off**: 柔軟性削減で controllability/verifiability を獲得

**focus-you への直結**:  
→ `Claude Code` の hook architecture および `/company` の部署パイプライン設計が「implicit 関係→explicit DAG」に既に移行。SGH の scheduler 理論を参考に、HD 内の task dependency を **formal DAG** として定義化すれば、社長の「プロジェクト進捗が不透明」という悩みを解決できる。具体的には `pipeline_state` テーブルに DAG structure を JSON で永続化し、dashboard の Pipeline View で可視化する。

---

### **(2026-04-14) GAM: Hierarchical Graph-based Agentic Memory**

**arXiv**: https://arxiv.org/abs/2604.12285 (v1: 2026-04-14)

**要点**:
- **記憶の分離**: "acquiring new information" と "retaining prior knowledge" のバランス困難を、encoding と consolidation を decoupling することで解決
- **グラフ構造**:
  - Event progression graph：ongoing dialogue を隔離
  - Topic associative network：semantic shift 時のみ統合
  - Graph-guided multi-factor retrieval
- **性能**: LoCoMo・LongDialQA で state-of-the-art。reasoning accuracy + efficiency 両立

**focus-you への直結**:  
→ **前回レポートで既出** の同一テーマだが、今週さらに 2論文が同じ方向性で投稿。記憶管理フレームワークは「LLM agent の critical bottleneck」として学術界で consensus。focus-you の diary + secretary_notes を GAM の「event ↔ topic」分層で再設計することで、感情 narrative の coherence を劇的に改善可能。

---

### **(2026-04-08) Program Analysis Guided LLM Agent for Proof-of-Concept Generation (PAGENT)**

**arXiv**: https://arxiv.org/abs/2604.07624 (v1: 2026-04-08)

**要点**:
- **用途**: Vulnerability PoC 自動生成
- **アプローチ**: 軽量 static analysis + sanitizer-based profiling + coverage 情報を LLM agent に feed
- **性能**: 従来 agentic approach より **132% 高性能**
- **革新性**: Pure LLM ではなく「analysis guidance + dynamic feedback ループ」が critical

**focus-you への含意**:  
→ 直接適用性は低い（脆弱性検査向け）。ただし、LLM agent の意思決定を external analysis（static/dynamic）で augment する pattern は、`polaris-circuit` の「回路仕様→部品推薦」エージェント設計に転用可能。PCB design constraint を static analysis（DRC / thermal simulation 等）で表現すれば、部品推薦精度向上。
