### [Externalization in LLM Agents: A Unified Review of Memory, Skills, Protocols and Harness Engineering](https://arxiv.org/html/2604.08224v1)

**(2604.08224, arXiv, 2026-04)**

**概要**: Memory, Skills, Protocols, Harness Engineering を「externalization（外部化）」の統一視点で分析する survey 論文。

**主要知見**:
- Agent capability = model + externalized artifacts (memory, skill, protocol)
- Memory: from implicit (embeddings) → explicit (structured text/graph)
- Skills: prompt-based vs. function-based vs. MCP server-based の進化
- Harness: session management, state persistence, recovery

**focus-you / HD 運営への示唆**:
- focus-you の「Diary → Memory Intelligence Agent → structured insight」アプローチは "memory externalization" の実装例。最新トレンドと合致
- HD 運営の Hook / MCP / skill の分業は "harness engineering" の実例。CLAUDE.md の責務分離が業界標準化しつつある

**推奨アクション**: なし（現在の設計が最先端）

### [A Survey on the Security of Long-Term Memory in LLM Agents: Toward Mnemonic Sovereignty](https://arxiv.org/html/2604.16548v1)

**(2604.16548, arXiv, 2026-04)**

**概要**: LLM Agent の long-term memory における attack surface, defense mechanisms, system architectures をカバーする新規 survey。

**セキュリティ脅威の分類**:
1. **Membership inference**: agent が「ユーザーの memory に特定データが含まれるか」を推測される
2. **Memory poisoning**: 悪意あるデータを memory に inject して agent の行動を hijack
3. **Privacy leakage**: memory が外部に流出
4. **Forgetting attacks**: 本来忘れるべき情報が長期保有される

**防衛メカニズム**:
- Memory compression（要約化で詳細情報削減）
- Differential privacy（統計的プライバシー保証）
- Federated learning（データ分散管理）
- Mnemonic Sovereignty（ユーザーが自分の memory を所有・制御）

**focus-you への直接的示唆**:
- focus-you の data portability 戦略（Markdown export + YAML frontmatter）は "Mnemonic Sovereignty" の実装。ユーザーが自分の日記・memory を export で持ち出せる設計が、セキュリティ論文で正当性確認
- **Diary データの暗号化**: edge encryption (client-side) 検討価値あり（memory poisoning 防止）

**推奨アクション**: 
- focus-you の privacy policy に「ユーザーが memory を完全削除・export 可能」を明記
- Markdown export に タイムスタンプ・integrity check (hash) を追加検討

### [MemMachine: A Ground-Truth-Preserving Memory System for Personalized AI Agents](https://arxiv.org/html/2604.04853v1)

**(2604.04853, arXiv, 2026-04)**

**概要**: Personalized AI agent の memory を raw episode (ground truth) と derived index (LLM extraction) の二層で管理するシステム。

**アーキテクチャ**:
- **Short-term memory**: current session context（prompt + output history）
- **Episodic memory**: raw conversation episodes（edit-safe, queryable）
- **Profile memory**: user attributes (preferences, behavior patterns)

**特徴**:
- Raw episode を保持 → LLM extraction errors の回復可能
- Edit history fully trackable
- Multi-turn conversation から facts を自動抽出しない（human-verified facts のみ記録）

**focus-you への示唆**:
- focus-you の「Diary」をこの three-tier memory structure に照らし合わせると：
  - Short-term: Today 画面（daily context）
  - Episodic: Diary entries（raw, editable）
  - Profile: Growth trends, themes, relationships（extracted, verified）
- 現在設計がほぼこの構造。Memory Intelligence Agent が episodic → profile 変換時に「raw source に立ち戻り検証」の仕組みを追加検討

**推奨アクション**: 小（HD 運営層では要検討、product feature は既に方向性正しい）

### [Spec2RTL-Agent: Automated Hardware Code Generation from Complex Specifications Using LLM Agent Systems](https://arxiv.org/html/2506.13905)

**(2506.13905, arXiv, 2026-05)**

**概要**: polaris-circuit に直接関連。複雑な仕様書から RTL (Register Transfer Language) コード生成を LLM agent で自動化。

**手法**:

- Code generation + simulation-based verification loop

**成果**:
- Complex specifications (1000+ lines) での生成成功率向上

**polaris-circuit への直接示唆**:

**推奨アクション**: 中（circuit チームが論文実装を参考に、agent-guided code gen pipeline を検討）
