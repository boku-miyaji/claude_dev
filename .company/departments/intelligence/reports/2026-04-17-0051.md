### (2026-04-16) Claude Opus 4.7 リリース — エージェント・コーディング・ビジョン大幅強化

**公式発表**: https://www.anthropic.com/news  
**モデルページ**: https://www.anthropic.com/claude/opus

Anthropic が 2026-04-16 に Claude Opus 4.7 を正式リリース。前世代(4.6)比で**コーディング、エージェント機能、ビジョン処理で大幅改善**。

**主要改善点**:
- **コーディング**: 従来は密接な監督が必要だった複雑なコーディングタスクを自律実行可能に
- **ビジョン（画像理解）**: 解像度が長辺 **2,576 ピクセル** に対応（従来の3倍以上）。高精度なドキュメント解析・図表認識に強化
- **エージェント**: マルチステップタスクの実行精度と「一貫性」が向上
- **信頼性**: 複雑な業務での thorough さと consistency を確保

**価格・利用可**: 価格は 4.6 と変わらず（$5/M input tokens, $25/M output tokens）。Claude Pro/Max/Team/Enterprise 全プランで利用可。Claude Platform / Amazon Bedrock / Google Vertex AI / Microsoft Azure でも利用可。

**focus-you への示唆**:  
→ **HD エージェント品質向上、自動化度の拡大が可能に**。特に Opus 4.7 のマルチステップタスク精度向上は、複数部署委譲時の「自律完遂率」改善に直結。ダッシュボード分析機能（感情分析、ナラティブ生成）の精度も向上する可能性。

### (2026-04-14) Claude Code Routines リリース（研究プレビュー） — クラウド自動化

**公式アナウンス**: https://claude.com/blog/introducing-routines-in-claude-code  
**ドキュメント**: https://code.claude.com/docs/en/routines  
**ベストプラクティス**: https://venturebeat.com/orchestration/we-tested-anthropics-redesigned-claude-code-desktop-app-and-routines-heres-what-enterprises-should-know

Claude Code に **3種類のルーティン** を搭載：

#### 1. Scheduled Routines
- **時間ベース実行**: 毎時、毎日、平日のみ、毎週
- CLI の `/schedule` コマンドで設定
- ローカル task 内の schedule と異なり、クラウド上で実行される

#### 2. API Routines
- **HTTP エンドポイント提供**: Routine を HTTP POST で外部システムから呼び出し可能
- アラーティングツール、監視システムから直接トリガー可能
- 例: Datadog / New Relic / Grafana からのアラート → Claude Code API Routine 自動実行

#### 3. Webhook Routines (GitHub)
- **リポジトリイベント購読**: GitHub Actions の次世代形態
- Pull Request イベント（作成、更新、コメント等）をトリガー
- **フィルター対応**: PR 作者、タイトル、本文、ブランチ、ラベル、draft 状態、fork 有無で条件指定可
- Claude GitHub App をインストール必須

**利用上限**:
- Pro: 5/日
- Max: 15/日  
- Team/Enterprise: 25/日

**リリース日**: 2026-04-14（研究プレビュー、全有料プランで利用可）

**focus-you への示唆**:  
→ **HD 秘書のバッチ処理・定期更新タスクが自動化可能に**。例えば：
- **朝ブリーフィング**: 毎朝9時に Scheduled Routine で情報収集レポートを自動生成
- **GitHub 連携**: PR がマージされたら自動で社長に通知・分析を Webhook Routine で実行
- **外部監視システム連携**: Socket.dev（セキュリティ脅威検知）→ API Routine で alert を即座に分析
- 現在の `GitHub Actions + Hook` ベースの構成を拡張でき、HD の反応速度が飛躍的に向上

## 🔴 セキュリティ警報：TeamPCP 供給チェーン攻撃キャンペーン

**参考**: 前回レポート後（2026-04-14以降）に詳細確認。**ただし攻撃発生は 2026-03-19 からのため、本レポートに含める**。

### 攻撃の概要

北朝鮮系グループ **UNC1069** / **BlueNoroff** が 2026-03-19～04-03 に大規模サプライチェーン攻撃を実行。npm と PyPI に合計 1,700+ の悪意あるパッケージをばら撒き。

### 主な被害（確認済み）

#### Axios npm パッケージ (2026-03-31)
- **被害**: npm 上の公式 Axios パッケージが直接侵害（100M+ downloads/week）
- **攻撃者**: メンテナー認証情報を窃取 → 2つの毒パッケージバージョンをリリース
- **ペイロード**: 隠れた悪意あるメガン、インストール直後に認証情報盗聴 + Remote Access Trojan 設置
- **検出**: Google が UNC1069 属性化公開（2026-04-16）

**参照**: https://thehackernews.com/2026/04/google-attributes-axios-npm-supply.html

#### LiteLLM (PyPI, 2026-03-24)
- バージョン 1.82.7, 1.82.8 で悪意あるコード注入
- PyPI が検知して隔離（quarantine）

#### Trivy、Telnyx 等（複数ベクトル）
- npm + PyPI 複数パッケージが同期的に侵害

### 攻撃パターンの特徴

1. **信頼の乗っ取り**: 偽パッケージでなく、**公式プロジェクトのメンテナー認証を侵害**（より危険）
2. **マルチベクトル**: npm, PyPI, Go, Rust エコシステムに同時侵害
3. **段階的リリース**: 2026-02-27 から 03-22 の間に計画的に投下
4. **属性**: 北朝鮮政府支援（Google、NSA 指摘）

### focus-you への示唆

→ **HD が依存する Python / npm パッケージの供給チェーンリスク再検証が必須。**
- 現在の dependencies（Supabase SDK, Anthropic SDK, playwright 等）の最新セキュリティステータスを確認
- `npm audit` / `pip-audit` の定期実行（Hook として組み込み検討）
- OAuth 認証・多要素認証を使用していないメンテナーアカウント（プロジェクト中）の強化推奨

**実行優先度**: 高（金融・医療領域ではないが、データハンドラー的性質）
