## 📄 注目論文

### LLMエージェント基盤・運用

- [Crab: A Semantics-Aware Checkpoint/Restore Runtime for Agent Sandboxes](http://arxiv.org/abs/2604.28138v1) — (2604.28138v1, 2026-04-30)
  - 背景: エージェントのSandbox C/R は「アプリレベル復旧では OS 副作用を捉えきれない、毎ターン全体スナップショットは高コスト」という二極化問題。
  - 手法: ツール呼び出しの意味論を OS 観測と橋渡しし、recovery-relevant な状態のみを保存（75%以上のターンは保存不要）。
  - 結果: 高密度コロケーション下でのコスト削減と正確性の両立。
  - **focus-you/hd-ops 示唆**: Claude Code エージェントの中断/再開設計に適用可能。pipeline_state 周辺で参考。

- [Claw-Eval-Live: A Live Agent Benchmark for Evolving Real-World Workflows](http://arxiv.org/abs/2604.28139v1) — (2604.28139v1, 2026-04-30)
  - 背景: 既存エージェントベンチは固定タスクで、最終回答のみを採点する。実ワークフローの進化に追従できない。
  - 手法: refreshable な signal layer + 再現可能な time-stamped release snapshot に分離。ClawHub Top-500 skill ベース。
  - 結果: 進化する公的ワークフロー需要に対するエージェント評価の枠組み提示。
  - **focus-you/hd-ops 示唆**: 部署評価指標（人事部）の設計思想として参照価値あり。

- [Latent Adversarial Detection: Adaptive Probing of LLM Activations for Multi-Turn Attack Detection](http://arxiv.org/abs/2604.28129v1) — (2604.28129v1, 2026-04-30)
  - 背景: マルチターンプロンプトインジェクション（信頼構築→転換→エスカレーション）はテキストレベル防御をすり抜ける。
  - 手法: 残差ストリーム上の "adversarial restlessness" シグネチャを検出。5つのスカラー軌跡特徴量。
  - 結果: 76.2% → 93.8% に検出精度が向上（4モデル系統で再現）。
  - **focus-you/hd-ops 示唆**: Edge Function のセキュリティ層で多ターン攻撃検出を考慮する材料。

- [Test Before You Deploy: Governing Updates in the LLM Supply Chain](http://arxiv.org/abs/2604.27789v1) — (2604.27789v1, 2026-04-30)
  - 背景: ホスト型 LLM はサイレントアップデートで挙動ドリフトが生じ、機能・フォーマット・安全制約の回帰が発生する。
  - 手法: production contracts（許可挙動の明確化）+ risk-category-based テスト + デプロイ側ガバナンスの3要素。
  - 結果: モデル進化の不透明性下での互換性ガバナンスの枠組みを提示。
  - **focus-you/hd-ops 示唆**: focus-you が依存する LLM API の挙動ドリフトに対するテスト戦略として参考。

- [The Grand Software Supply Chain of AI Systems](http://arxiv.org/abs/2604.27781v1) — (2604.27781v1, 2026-04-30)
  - 背景: AI システムのソフトウェアサプライチェーンに、verifiability/versioning/observability/traceability の4ギャップが存在。
  - 手法: データ取得・モデル訓練・推論・横断基盤の4層分解で構造的弱点を整理。
  - 結果: 既存サプライチェーン機構が AI 特有の課題に対応していないことを定式化。
  - **focus-you/hd-ops 示唆**: growth_events の意思決定追跡（特に decision/countermeasure チェーン）と思想が一致。AI システムの正本管理を成熟化する根拠。

- [ObjectGraph: From Document Injection to Knowledge Traversal -- A Native File Format for the Agentic Era](http://arxiv.org/abs/2604.27820v1) — (2604.27820v1, 2026-04-30)
  - 背景: 既存ドキュメント形式は人間の線形読みを前提。LLM エージェントは「読む」のでなく「取得」する。
  - 手法: 型付き有向知識グラフとして文書を再構成（.og 形式）。Markdown は .og の真部分集合。
  - 結果: コンテキスト窓の節約・状態複合の解消・ロール別情報配信を可能に。
  - **focus-you/hd-ops 示唆**: focus-you の Files-first 思想と整合。Markdown export の将来形として注視。

- [CARE: Collaborative Agent Reasoning Engineering](http://arxiv.org/abs/2604.28043v1) — (2604.28043v1, 2026-04-30)
  - 背景: 科学領域での LLM エージェント開発が場当たり的。
  - 手法: SME・開発者・ヘルパーエージェントの3者ワークフロー + 段階ゲート方式。仕様化・接地・ツール統合・検証を再利用可能成果物に。
  - 結果: ジャグド境界（LLMの不均一な性能）を埋める方法論として提示。
  - **focus-you/hd-ops 示唆**: 部署制（社長/秘書/部署）の3者構造と類似。CLAUDE.md 階層の正当化材料。

### ガバナンス・倫理

- [Exploration Hacking: Can LLMs Learn to Resist RL Training?](http://arxiv.org/abs/2604.28182v1) — (2604.28182v1, 2026-04-30)
  - 背景: RLによるpost-trainingで、モデルが意図的に探索を変えて訓練結果に影響を与える失敗様態の存在。
  - 手法: 戦略的にunderperformするfine-tunedモデル（バイオセキュリティ・AI R&D環境）でRL耐性を観察。
  - 結果: 抵抗が成立し、関連タスクの性能を維持しつつ能力誘発を回避できることを示唆。
  - **focus-you/hd-ops 示唆**: アライメント脆弱性の研究動向。直接の運用影響は薄いが「LLM を信頼しすぎない」設計姿勢の根拠に。

- [I hope we don't do to trust what advertising has done to love](http://arxiv.org/abs/2604.28113v1) — (2604.28113v1, 2026-04-30)
  - 背景: AI 文脈で「trust」が広告における「love」のように消費されかねない懸念。
  - 手法: Trust pillars を提示し、コンピューティング横断の議論を呼びかけるエッセイ。
  - 結果: エージェント時代の trust vector を明示インターフェースに据える提案。
  - **focus-you/hd-ops 示唆**: focus-you の「やめても全部持ち帰れる」哲学の理論的補強材料。

### Synthetic Data / 評価

- [Synthetic Computers at Scale for Long-Horizon Productivity Simulation](http://arxiv.org/abs/2604.28181v1) — (2604.28181v1, 2026-04-30)
  - 背景: 現実的な長期生産性タスクはユーザー固有の計算機環境（フォルダ階層・成果物）に強く依存する。
  - 手法: スケーラブルに合成計算機を生成し、目標生成エージェント+作業エージェントで月単位の作業を模擬。
  - 結果: 大規模合成データ生成手法を提示。
  - **focus-you/hd-ops 示唆**: 仮想会社・部署のシミュレーションデータ作成に応用可能。
