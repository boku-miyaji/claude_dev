## 📄 注目論文（arXiv 2026-04-22〜23新着）

### 1. [GAM: Hierarchical Graph-based Agentic Memory for LLM Agents](https://arxiv.org/abs/2604.12285)

**(2604.12285, arXiv:2604.12285, 提出: 2026-04-14)**

**背景・目的**: LLM エージェントが長期対話を維持するとき、新情報の迅速な認識と既知情報の安定保持のバランスに葛藤が生じる。従来フラットなメモリ設計では、新情報が既知知識を上書き・干渉する。

**手法**: 階層的グラフベースメモリ（GAM）
- **イベント進行グラフ**: 進行中の対話をリアルタイムで分離
- **トピック関連ネットワーク**: セマンティックシフト時のみ統合（干渉最小化）
- 流ストリーム + 構造化メモリの融合設計

**主要結果**: 
- LoCoMo、LongDialQA ベンチマークで SOTA を上回る
- 推論精度と効率の両面で優越性確認
- **focus-you への応用可能性**: Diary + Memory Intelligence Agent の永続記憶アーキテクチャに直結

### 2. [From Agent Loops to Structured Graphs: A Scheduler-Theoretic Framework for LLM Agent Execution](https://arxiv.org/abs/2604.11378)

**(2604.11378, arXiv:2604.11378, 提出: 2026-04-13)**

**背景・目的**: 現在の「Agent Loop」パラダイム（再帰的な step → observe → act）には根本的な構造問題が存在する：
- ステップ間の暗黙的依存関係が隠蔽されている
- 無制限の回復ループ
- デバッグの複雑性

**手法**: Structured Graph Harness（SGH）
- 実行計画を版内で不変化（immutable DAG）
- 計画実行と回復を3層に分離
- 厳密なエスカレーションプロトコル導入
- 制御構造を「コンテキストの暗黙要素」から「明示的 static DAG」へ昇格

**主要結果**: 
- 70の調査対象システム分析
- 制御可能性・表現力・実装可能性のトレードオフ分析を提供
- **位置論文・設計提案として機能**（実装結果は future work）
- **HD運営への示唆**: `.claude/rules/pipeline.md` の DAG ベース部署計画が学術的に裏付けられた

### 3. [Drop the Hierarchy and Roles: How Self-Organizing LLM Agents Outperform Designed Structures](https://arxiv.org/abs/2603.28990)

**(2603.28990, arXiv:2603.28990, 掲載: 2026-03-30)**

**背景・目的**: 従来の マルチエージェント設計では「事前割り当て役割」「集中型調整」を前提としてきた。しかし LLM エージェントは、最小限の足場（固定順序のみ）で、自発的に role を生成・再構成できるか？

**手法**: 25,000タスク計算実験
- 8モデル（GPT, Claude, Gemini 等）
- 4〜256エージェント規模
- 8つの協調プロトコル比較（Sequential hybrid + Centralized + Decentralized）

**主要結果**:
- **Sequential hybrid が集中型を14%上回る**（p<0.001）
- **8エージェントで 5,006 の独自役割が自動生成される**
- **オープンソースモデルが24倍低コストで、クローズドソース品質の95%を達成**
- **256エージェントへのスケーリングで品質低下なし**（p=0.61）
- **focus-you への示唆**: Managed Agents + self-organization パターンの有効性が実証された

### 4. [Externalization in LLM Agents: A Unified Review of Memory, Skills, Protocols and Harness Engineering](https://arxiv.org/abs/2604.08224)

**(2604.08224, arXiv:2604.08224, 掲載: 2026-04 中旬)**

**背景・目的**: 2022〜2026 の4年間で、LLM agent の「能力の源」が劇的に変化した：
- **2022**: モデルの重み（weight）
- **2023-24**: プロンプト設計（prompt）
- **2025-26**: より広い基盤構造（tools, memory, protocols, harness）

この externalization トレンドを統一フレームワークで理論化。

**手法**: 時系列分析 + アーキテクチャレビュー
- Memory externalization（永続ストレージ化）
- Skill externalization（ツール / MCP へ）
- Protocol externalization（制御フロー定義）
- Harness externalization（実行環境・ガバナンス）

**主要結果**:
- 学術的に externalization の必然性を証明
- **HD architecture（Hook + Agent + Company）** が externalization framework に合致
- **宮路HD運営への示唆**: `.claude/rules/hd-operations.md` の Hook/Agent/Company 分離が「externalization」として正当化される

### 取り入れるべき

- **GAM（階層的グラフメモリ）パターンの即時導入検討**
  → Diary entry を単純なリスト構造から「event stream + topic network」に昇格。Memory Intelligence Agent の context-aware retrieval を直結。
  - 対象: `supabase/functions/ai-agent/memoryAgent.ts`, Diary テーブルスキーマ
  - 工数: 大（新テーブル設計、retrieval query 最適化）

- **Self-Organizing Agent パターンの実装参考（25,000タスク実証済み）**
  → focus-you の Narrators（複数エージェント）が「自発的に専門化する」メカニズムを論文が証実。Managed Agents + autonomous role emergence の組み合わせ価値確認。
  - 対象: `company-dashboard/src/pages/Blueprint.tsx` (Narrators tab)
  - 工数: 小〜中（既存アーキテクチャとの整合確認のみ）
  - 優先度: **中**（次フェーズの multi-narrator coordination）

### 検討に値する

- **Structured Graph Harness（SGH）アーキテクチャへの昇格検討**
  → 現在の「Agent Loop」（再帰的 observe-act）から「immutable DAG + explicit escalation」へ。focus-you のシステムプロンプト（`.claude/rules/pipeline.md`）がすでに DAG 志向だが、実装レベルで明示化する価値あり。
  - 判断ポイント: Agent の failure case が escalation を必要としているか（ユーザーアンケート）

- **Claude Code の theme customization を dashboard に統合**
  → v2.1.118 の カスタムテーマ機能を、focus-you dashboard のセッション視覚化に応用。ユーザーが「自分の感情と matched な色スキーム」を選択可能化。
  - 判断ポイント: Diary mood + dashboard theme の連動が認知負荷を減らすか（UX 検証必要）

### 現状で正しいと確認できたもの

- **Memory を最重要な externalization 対象として位置づけていること**
  → GAM、Cognis、SkillLearnBench など複数論文が「agent capability = memory architecture」を証実。focus-you の Diary + Memory Intelligence Agent 設計が trend ahead。

- **Managed Agents + 秘書による判断ゲートの二層構造**
  → 論文「Self-Organizing Agents」で「最小限の足場（秘書の DAG）+ autonomous role emergence（agent 自己組織化）」が最適と実証。focus-you の `.company/CLAUDE.md` 設計が学術的に正当化された。

- **MCP を「externalization の標準インターフェース」として採用していること**
  → Externalization レビュー論文で「skills / protocols / harness の externalization」が必須と論理化。focus-you の tools / hooks / plugins が MCP 化対象として優先度 **高** に設定可能。

  - 工数: 小（既存設計の理論化のみ）
  - 優先度: **中**（後進者への説得力向上）

- **Pipeline DAG の「immutable + explicit escalation」化**
  → Structured Graph Harness 論文が Agent Loop パラダイムの問題点を指摘。focus-you の `.claude/rules/pipeline.md` で「DAG versioning」「escalation protocol」を明文化し、複雑な部署協業の失敗を予防。
  - 対象: `pipeline.md` + Agent tool の failure handling
  - 工数: 中（実装と検証）
  - 優先度: **高**（現在の pipeline が複雑化している）

- **MCP を「externalization インターフェース」として positional update**
  → Agentic AI Foundation（10,000+ servers）と Externalization 論文で、MCP が「skills / protocols のデ facto standard」に昇格。focus-you の「Hook → MCP tool」化を official requirement に格上げ。
  - 対象: `.company/mcp-profiles.yaml`, MCP server 新規作成ガイドライン
  - 工数: 中（MCP spec learning + POC）
  - 優先度: **中**（次 quarter の focus）

### 検討に値する

- **Agent failure recovery の escalation protocol 形式化**
  → SGH 論文で「recovery の3層分離」「escalation step」が提案されている。focus-you のエラーハンドリング（`.claude/hooks/` 内の例外検出）を escalation framework に再設計する価値あり。
  - 判断ポイント: 現在、agent が fail したときの秘書 → 社長への escalation が明示的か（logs を確認）

- **Growth Events の「DAG化」**
  → `growth_events` テーブルが現在 parent_id ベースだが、SGH の「immutable DAG」パターンで versioning を追加。「failure A → countermeasure A1 → superseded by A2」の履歴が explicit に追跡可能化。
  - 判断ポイント: 過去の対策が「なぜ supersede されたか」を後発者が理解できるか

### 現状で正しいと確認できたもの

- **Hook による「軽量・非同期記録」と company による「判断」の分離**
  → Externalization 論文で「capability externalization」として位置づけられた。`.claude/rules/hd-operations.md` の Hook/Agent/Company 分離が **学術的根拠あり**。

- **Pipeline を DAG で表現していること**
  → Structured Graph Harness 論文で「implicit dependencies を explicit DAG に昇格」が推奨。focus-you の `pipeline.md` がすでに DAG ベース表記。継続でOK。
