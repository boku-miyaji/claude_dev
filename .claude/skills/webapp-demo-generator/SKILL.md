# Web App Demo Generator

このスキルは、Webアプリ向けのデモ動画生成を支援するためのものです。
以下のような依頼で使います。

- 「このWebアプリのデモ動画を自動生成したい」
- 「Playwrightで操作を録画して字幕も付けたい」
- 「Claude Codeでデモ生成パイプラインを作りたい」
- 「字幕、ナレーション、ffmpeg合成まで含めた構成を作りたい」
- 「PRごとにデモ動画を生成したい」

## このスキルがやること

ユーザーの要望に応じて、以下を行う。

### 1. デモ動画化の対象を整理

- 対象Webアプリ
- 想定ユーザー
- 見せたい業務価値
- デモ対象機能
- 入力データや前提条件
- ローカル環境 / ステージング環境 / 本番相当環境

### 2. デモ構成を設計

- どの操作をどの順番で見せるか
- 1本の動画にするか、機能ごとに分けるか
- 録画方式
- 字幕生成方式
- ナレーション生成方式
- 合成方式

### 3. 必要な成果物を生成

- Playwrightスクリプト
- デモ台本
- 字幕原稿（SRTまたはVTT）
- ナレーション原稿
- ffmpegコマンド
- ディレクトリ構成案
- npm scripts
- GitHub Actions案

### 4. 実装時の注意点を整理

- 認証
- テストデータ
- シークレット管理
- 表示待機
- アニメーションやローディング対策
- 画面サイズ固定
- 音声・字幕タイミング
- 個人情報マスキング

## 基本方針

- まずは**Webアプリに限定**して考える
- ブラウザ操作は基本的に **Playwright** を第一候補とする
- 録画・字幕・ナレーション・動画合成を分離して設計する
- いきなり巨大実装にせず、まずは最小構成を提示する
- 実運用を意識して、ローカル実行版とCI版の両方を考える
- ユーザーが要件を十分に出していなくても、妥当な前提を置いて進める
- 不明点があっても、止まらず「仮定」を明示して出す

## 推奨アーキテクチャ

標準構成は以下。

1. Playwrightでブラウザ操作
2. Playwrightまたは外部手段で録画
3. 操作ログから字幕原稿を生成
4. 必要ならTTSでナレーション音声を生成
5. ffmpegで字幕・音声・動画を合成

## 推奨出力フォーマット

ユーザーへの回答は、できるだけ以下の順で整理する。

### 1. 推奨構成

- 何を使うか
- なぜそれが適切か

### 2. ディレクトリ構成

- どこに何を置くか

### 3. 実装コード

- Playwright script
- subtitle generator
- narration script
- ffmpeg command
- package.json scripts

### 4. 実行手順

- install
- record
- subtitle generate
- voice generate
- mux

### 5. 拡張案

- GitHub Actions
- PRごとの自動生成
- 複数シナリオ化
- ストーリー分岐
- インタラクティブデモ化

## 実装のデフォルト前提

ユーザー指定がなければ以下を仮定してよい。

- Node.js 20系
- TypeScript利用可
- Playwright採用
- 動画は mp4
- 字幕は srt
- ローカル環境で一旦動かす
- 画面サイズは 1440x900 前後
- 対象はログイン後の業務画面
- ナレーションは任意
- まずは1シナリオのみ

## 出力時の注意

- 可能な限り、そのまま動かせるコードを出す
- ダミーURLやセレクタを使う場合は、置き換え箇所を明記する
- 認証情報は直書きしない
- 個人情報や実データの露出リスクを指摘する
- UI要素の待機は `waitForLoadState` や `locator(...).waitFor()` を使い、雑な sleep を乱用しない
- 録画の不安定性や headless/headed の差を説明する
- 実装が重い場合でも、最低限のMVP版を先に出す
