## 1. シナリオ分析

### プライマリシナリオ: 自己理解の定期チェック

```
誰:   社長（複数PJ並行管理するフリーランス/AI開発者）
いつ:  週末の夜、または振り返りの時間に
どこ:  PC（メイン）またはスマホ
何を:  「自分の性格傾向、強み、価値観」を客観的に把握したい
感情:  好奇心 + 少しの不安（「自分を知る」ことへの期待と緊張）
目的:  意思決定の軸を確認する。キャリアの方向性を検証する。
```

### セカンダリシナリオ: 変化の追跡

```
誰:   同じユーザー（2回目以降の訪問）
いつ:  月1回程度、新しい日記データが蓄積された後
何を:  「前回から自分はどう変わったか」を知りたい
感情:  成長への期待。変化がないことへの焦り。
目的:  行動を変えた結果がパーソナリティに反映されているか確認。
```

### ユーザーの本質的な欲求

データを見たいのではなく、**「自分を一言で理解できるストーリー」**が欲しい。
スコアの数字そのものより、「それが何を意味するか」「どう活かせるか」を知りたい。

## 2. 感情曲線とフリクションポイント

```
[ページを開く] ............... 期待感 (+)
  |
  v
[全体サマリーが目に入る] ..... 安心感 (++) 「一目で全体像がわかる」
  |
  v
[気になる分析をタップ] ....... 好奇心 (+++)
  |
  v
[詳細テキストが長い] ......... 認知負荷 (-) <-- FP1: テキストの壁
  |
  v
[スコアの意味がわからない] ... 困惑 (--) <-- FP2: 数字だけでは伝わらない
  |
  v
[前回との変化が見つかる] ..... 発見 (++) 
  |
  v
[「次にどうすればいい？」] ... 行動への迷い (-) <-- FP3: 出口がない
```

### フリクションポイント一覧

| ID | ポイント | 現状の問題 | 影響度 |
|----|---------|-----------|--------|
| FP1 | テキストの壁 | 500字以上の description が未構造化で縦に流れる | 高 |
| FP2 | 数字の意味不明 | スコアバーだけで「75点」が良いのか悪いのかわからない | 高 |
| FP3 | 出口なし | 分析結果を見た後の「次のアクション」がない | 中 |
| FP4 | 全体像不在 | 4分析がカードで縦に並び、スクロールしないと全体が見えない | 高 |
| FP5 | 視覚的退屈 | 横棒グラフしかなく、見る楽しさがない | 中 |

## 3. 設計判断ログ

### FP1 対策: テキストの壁 -> 構造化カード

```
適用原則: #2 認知負荷の最小化（ミラーの法則: 7±2チャンク）
設計判断: 長文テキストを「特徴」「活かし方」「注意点」のカードに分割
見送案:   アコーディオンで折りたたむ → 情報が隠れすぎる
```

### FP2 対策: 数字の意味不明 -> ビジュアル + ラベル

```
適用原則: #3 フィードバックループ（フィードフォワード）
設計判断: スコアに「高い/平均/低い」のラベルと色を付与。
          Big5はペンタゴンレーダーで形で把握。MBTIは中央起点の双方向バー。
見送案:   スコアを非表示にして文章だけ → 定量的な把握ができない
```

### FP3 対策: 出口なし -> アクション提案セクション

```
適用原則: #1 意図の先読み（Anticipatory Design）
設計判断: 各分析の末尾に「活かし方」を箇条書きで表示。
          SF分析には work_fit タグを目立たせる。
見送案:   外部リンクで性格タイプの解説サイトに飛ばす → フロー切断
```

### FP4 対策: 全体像不在 -> ダッシュボードビュー + 詳細タブ

```
適用原則: #2 認知負荷の最小化 + #5 フロー状態の保護
設計判断: 上部に4分析のコンパクトサマリーを1スクリーンに。
          詳細はタブ切り替えで同一ページ内に。
見送案:   4分析を別ページに分ける → コンテキストスイッチが多すぎる
```

### FP5 対策: 視覚的退屈 -> SVGチャート

```
適用原則: #3 フィードバック（感情のフィードバック: 美しさによる満足感）
設計判断: Big5にペンタゴンレーダー、MBTIに双方向バー、
          SFに4ドメインドーナツ、Valuesにバブルランキング
見送案:   外部チャートライブラリ → 依存増加、バンドルサイズ
```

## 5. 状態遷移設計

### 5.1 ページ全体の状態

| 状態 | UI表現 | トリガー |
|------|--------|---------|
| **loading** | スケルトン表示（ダッシュボード4枠のプレースホルダー） | ページ初回アクセス |
| **empty** | イラスト + 「日記を20件以上書いて分析を始めましょう」+ 日記ページへのリンク | データ0件 or 日記<20件 |
| **loaded** | ダッシュボード + タブ付き詳細 | データ取得完了 |
| **stale** | 「最終分析: X日前 / 新しい日記がN件あります」バッジ | 分析後に新規日記あり |
| **analyzing** | 進捗ステッパー（MBTI -> Big5 -> SF -> Values の4ステップ表示） | 分析実行中 |
| **error** | インラインエラーバナー + リトライボタン | API失敗 |
| **partial** | 完了した分析のみ表示 + 残りはスケルトン | 分析途中 |

## 6. インタラクション仕様

### 6.1 タブ切り替え

- **アニメーション**: タブインジケーター（下線）が accent色でスライド移動（transition: left 0.2s ease-out）
- **コンテンツ**: フェードイン（opacity 0->1, 150ms）。スライドはしない（認知負荷軽減）
- **キーボード**: 左右矢印キーでタブ移動、Enter/Spaceでアクティブ化
- **ARIA**: role="tablist" / role="tab" / role="tabpanel" / aria-selected

### 6.2 ダッシュボードカードのホバー

- **PC**: カードホバーで微かなリフト（translateY(-2px)、shadow-lg）+ border-color変化
- **クリック**: 対応するタブを開く（スムーズスクロール + タブアクティブ化）
- **モバイル**: タップで即座にタブ切り替え

### 6.3 SVGチャートのアニメーション

- **ペンタゴン（Big5）**: ページ表示時に中心から展開（scale 0->1、300ms ease-out）
- **双方向バー（MBTI）**: 中央から左右に伸びる（width 0->target、200ms per bar、50ms stagger）
- **ドーナツ（SF）**: 0度から時計回りに描画（stroke-dashoffset アニメーション、500ms）

### 6.4 分析実行フロー

1. ボタンクリック -> ボタンがスピナーに変化
2. ステッパーが表示される（ダッシュボードカードは薄くなる）
3. 各分析が完了するたびに対応するカードが更新される（楽観的UIではなく実データ反映）
4. 全完了 -> ステッパーがフェードアウト -> ダッシュボード完全表示
5. トースト通知: 「分析が完了しました」（3秒で自動消去）

### 7.2 SVGチャート実装仕様

#### Big5 ペンタゴンレーダーチャート

```typescript
/**
 * ペンタゴンレーダーチャート（Big5用）
 * 
 * @param scores - { openness, conscientiousness, extraversion, agreeableness, neuroticism } 各0-100
 * @param size - SVGのwidth/height（デフォルト: 200）
 * @param animated - 表示アニメーション有無
 */
interface PentagonRadarProps {
  scores: {
    openness: number          // 頂点（12時方向）
    conscientiousness: number // 右上（1-2時方向）
    extraversion: number      // 右下（4-5時方向）
    agreeableness: number     // 左下（7-8時方向）
    neuroticism: number       // 左上（10-11時方向）
  }
  size?: number
  animated?: boolean
}

// 計算ロジック:
// 5頂点の角度: -90, -90+72, -90+144, -90+216, -90+288 (度)
// 各頂点の座標: center + (score/100) * radius * cos/sin(angle)
// 
// 描画レイヤー:
// 1. 背景グリッド: 25%, 50%, 75%, 100% の同心ペンタゴン（stroke: var(--surface2)）
// 2. データポリゴン: 塗りつぶし rgba(accent, 0.15) + stroke accent 2px
// 3. 各頂点にドット（r=4, fill: accent）
// 4. ラベル: 各頂点の外側にテキスト + スコア値
//
// 色:
//   グリッド線: var(--border) opacity 0.3
//   データ塗り: var(--accent) opacity 0.15
//   データ線: var(--accent) opacity 0.8
//   頂点ドット: var(--accent)
//   ラベル: var(--text2), スコア: var(--accent2) mono
```

#### MBTI 双方向バー

```typescript
/**
 * MBTI 4次元の双方向バーチャート
 * 中央(0)を起点に、スコアに応じて左右に伸びる
 * 
 * @param dimensions - { E_I, S_N, T_F, J_P } 各 { score: -100~100, label }
 */
interface MbtiDimensionBarProps {
  dimensions: Record<string, { score: number; label: string }>
}

// 各バーの構造:
// [左ラベル] [=======|████████████>] [右ラベル]
//            -100     0    score     +100
//
// 中央に薄い縦線（var(--border)）
// スコアが正: 中央から右に accent色で塗る
// スコアが負: 中央から左に accent色で塗る
// バーの高さ: 8px, border-radius: 4px
// 背景: var(--surface2)
// 各バー間の間隔: 16px
//
// ラベルの色分け:
//   活性側: var(--text), fontWeight 600
//   非活性側: var(--text3), fontWeight 400
```

#### SF 4ドメインドーナツチャート

```typescript
/**
 * ストレングスファインダー 4ドメインバランスのドーナツチャート
 * 
 * @param domains - { strategic_thinking, relationship_building, influencing, executing }
 */
interface DomainDonutProps {
  domains: Record<string, { score: number; label: string }>
}

// SVG circle の stroke-dasharray/stroke-dashoffset で実装
// 4セグメントの色:
//   strategic_thinking: var(--accent)  (戦略的思考力)
//   relationship_building: var(--green) (人間関係構築力)
//   influencing: var(--amber)          (影響力)
//   executing: var(--red)              (実行力)
//
// 中央に最もスコアが高い領域名を表示
// 外周に各セグメントのラベル + スコア
// strokeWidth: 20, radius: 60
```

### 7.3 レスポンシブ設計

```
Desktop (>768px):
  ダッシュボード: 2x2 グリッド (g2 クラス活用)
  詳細ビュー: 全幅

Mobile (<=768px):
  ダッシュボード: 1列スタック
  タブ: 横スクロール可能 (overflow-x: auto, -webkit-overflow-scrolling: touch)
  SVGチャート: size を縮小 (200px -> 160px)
  引用カード: 横パディング縮小
```

### 7.4 アクセシビリティ (WCAG 2.1 AA)

| 要件 | 対応 |
|------|------|
| タブのキーボード操作 | role="tablist", arrow key navigation |
| SVGチャートの代替テキスト | aria-label に数値情報を含める |
| カラーコントラスト | テキストは 4.5:1 以上。グラフは色+形で区別 |
| スクリーンリーダー | 各スコアに sr-only テキストで「高い/中程度/低い」を付与 |
| フォーカス表示 | focus-visible で accent色のリング |
| 動作軽減 | prefers-reduced-motion でアニメーション無効化 |

### 7.5 データフロー

```
1. ページマウント
   -> load() : diary + tasks + dreams フェッチ + self_analysis 最新取得
   -> pastResults に格納
   -> latestByType() で各タイプの最新を抽出

2. ダッシュボード表示
   -> pastResults が空なら EmptyState
   -> あれば DashboardSummary に result を渡す

3. タブ切り替え
   -> useState<AnalysisType>('mbti') でアクティブタブ管理
   -> 対応する Detail コンポーネントをレンダリング

4. 再分析実行
   -> handleRunAll() : ALL_TYPES を順次実行
   -> 各完了時に pastResults を更新（即座にUIに反映）
   -> AnalysisProgress で進捗表示
```

## 8. 実装優先度

| 優先度 | 項目 | 理由 |
|--------|------|------|
| P0 | ダッシュボード+タブ構造 | 全体像把握が最大のペインポイント |
| P0 | MBTI双方向バー | 既存のバーを改良するだけで実装コスト低 |
| P1 | Big5ペンタゴンSVG | 視覚的インパクト最大。「見たくなる」の核 |
| P1 | テキスト構造化カード | 認知負荷の改善に直結 |
| P2 | SFドーナツチャート | あると良いが、バー表示でも機能する |
| P2 | 分析進捗ステッパー | UX改善だが必須ではない |
| P3 | アニメーション | 体験の磨き込み。後からでも追加可能 |
| P3 | Staleバッジ | 再訪率向上。データ追加後に効く |
