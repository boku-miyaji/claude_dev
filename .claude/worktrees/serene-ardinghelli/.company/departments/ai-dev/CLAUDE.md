# AI開発部署（HD共通）

## 役割
LLM/AIシステムの要件定義・設計・実装・評価・運用を一貫して担当する。
PJ会社のCLAUDE.mdに記載されたリポジトリ・技術スタック・ドメインに従って作業する。

## チーム構成

| チーム | フォルダ | 役割 |
|--------|---------|------|
| 要件定義 | requirements/ | ユースケース整理、AI適用可否判断、要件整理 |
| 設計 | design/ | アーキテクチャ設計、プロンプト設計、モデル選定 |
| 実装 | implementation/ | LLM API連携、RAG構築、エージェント実装 |
| アルゴ | algorithm/ | アルゴリズム検討、最適化、モジュール化 |
| 評価 | evaluation/ | 精度評価、ベンチマーク、品質保証 |
| AIOps | aiops/ | デプロイ、モニタリング、運用 |

## 作業サイクル（IMPORTANT）

AI開発は「一直線に作って終わり」ではない。以下のサイクルを回す:

```
仮説設定 → 設計 → [設計レビュー] → 実装 → 評価 → [品質判定]
   ↑                                              │
   └──── 学習（なぜ失敗/成功したか記録）←──────────┘
```

### Step 1: 仮説設定（要件定義チーム）
- ユーザーが何を達成したいか、AIでどう解決するかの仮説を立てる
- **完了条件**: ユースケース文書 + 成功基準（定量指標）が定義されている
- 出力: `requirements/YYYY-MM-DD-task-name.md`

### Step 2: 設計（設計チーム）
- アーキテクチャ、プロンプト構成、モデル選定、データフロー
- **完了条件**: 実装チームが「これだけ読めば作れる」レベルの仕様書
- 出力: `design/YYYY-MM-DD-task-name.md`

### Step 3: 設計レビュー（チェックポイント）
- 設計書を秘書に提出。checkpointモードなら社長確認。
- レビュー観点:
  - [ ] 成功基準が定量的か（「良い感じ」は不可。「精度80%以上」等）
  - [ ] 既存のコードパターンとの整合性
  - [ ] 外部依存（API、モデル）のコスト・レート制限の確認
- **差し戻し条件**: 成功基準が曖昧 / 既存パターンと矛盾

### Step 4: 実装（実装+アルゴチーム）
- 設計書に忠実に実装。プロンプトは設計チームの管轄（勝手に変更しない）
- **完了条件**: コードがコミット済み + 動作確認済み
- 出力: 実コード（PJリポジトリ） + `implementation/YYYY-MM-DD-task-name.md`（実装メモ）

### Step 5: 評価（評価チーム）
- Step 1 で定義した成功基準に対して定量評価
- **完了条件**: 評価レポートに数値結果 + 合格/不合格の判定
- 出力: `evaluation/YYYY-MM-DD-task-name.md`

### Step 6: 品質判定（チェックポイント）
- 評価結果を確認:
  - **合格** → AIOps（デプロイ）or 完了報告
  - **不合格** → Step 1 に戻る（仮説の修正 or 設計の見直し）
  - **部分合格** → 不合格項目のみ再設計→再実装→再評価

### Step 7: 学習（毎回）
- 成功/失敗に関わらず、得られた知見を記録
- 出力: 成果物末尾に「## 学習メモ」セクション
- 重要な発見は `growth_events` に記録

## スキップルール

全ステップを毎回経由する必要はない。ただし省略の判断基準を明示:

| タスク規模 | 省略可能 | 必須 |
|-----------|---------|------|
| 1行修正 | 要件・設計・評価 | 実装 + 動作確認 |
| 小改善（既存の延長） | 要件 | 設計 + 実装 + 動作確認 |
| 新機能 | なし | 全ステップ必須 |
| アーキテクチャ変更 | なし | 全ステップ + 社長レビュー必須 |

## ルール
- 外部（sys-dev等）への提供物は必ずAPI仕様/モジュール仕様を明記する
- 評価チームを経由せず本番投入しない（1行修正を除く）
- プロンプトは設計チームが管理。実装チームが勝手に変更しない
- アルゴチームの成果物は必ずモジュール化し、依頼元が使えるインターフェースで提供
- 各チームのファイルは `チームフォルダ/kebab-case-title.md`
