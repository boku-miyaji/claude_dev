## 2. 画面設計（モバイルファースト）

### Self-Analysis（自己分析）— AIが「あなたはこういう人です」を言語化する

**ビジョンの核心機能。** diary・行動データ・プロンプトログからAIが多角的に分析し、自分では気づかない傾向を言語化する。

```
蓄積データ                       分析カテゴリ
├── diary（感情・内省）     ──→  性格分析（MBTI、Big5）
├── プロンプトログ（思考）  ──→  思考スタイル分析
├── タスク完了パターン      ──→  仕事適合性・強み分析
├── 感情の時系列           ──→  感情パターン分析
├── 夢・目標の選び方       ──→  価値観分析
└── 行動の時間帯・曜日     ──→  行動パターン・リズム分析
```

**分析メニュー（段階的にアンロック）:**

| 分析 | 必要データ量 | 出力例 |
|---|---|---|
| **MBTI推定** | diary 20件〜 | 「日記の傾向からINTJ-Tに近い思考パターンが見られます。特に...」 |
| **Big5パーソナリティ** | diary 30件〜 | 開放性・誠実性・外向性・協調性・神経症傾向のスコアと根拠 |
| **強み・才能分析** | タスク50件〜 | 「設計タスクの完了速度が実装の2倍。構造化思考が強みです」 |
| **仕事適合性** | プロンプト50件〜 | 「技術調査→設計の流れで最も集中力が高い。PM的タスクは後回し傾向」 |
| **感情トリガーマップ** | diary 30件 + 感情データ | 「MTG後に不安が上がる傾向。一人作業後に喜びが上がる」 |
| **価値観の優先順位** | diary 50件 + 夢リスト | 「成長 > 自由 > 安定 > 承認。この3ヶ月で『自由』の重要度が上昇」 |
| **行動リズム分析** | プロンプト30件〜 | 「10-12時が最も生産的。木曜午後に判断力が低下する傾向」 |
| **ストレス耐性プロファイル** | diary 40件 + 感情データ | 「締切プレッシャーには強いが、曖昧な状況への不安が高い」 |
| **成長軌跡** | 3ヶ月以上の蓄積 | 「3ヶ月前と比べて意思決定の速度が1.4倍。不安の頻度が20%減少」 |
| **コミュニケーションスタイル** | プロンプト50件〜 | 「簡潔で本質重視。詳細説明より構造化された情報を好む」 |

**分析の特徴:**
- **全てデータ駆動**: 質問票ではなく、実際の行動・感情データから推定
- **根拠を明示**: 「なぜそう分析したか」を具体的なdiary引用やデータで示す
- **変化を追跡**: 同じ分析を定期的に再実行し、自分の変化が見える
- **言語化の支援**: 「自分はこういう人間なのかも」を言葉にするのを手伝う

## 3. データアーキテクチャ

### AIの理解モデル

```
蓄積データ（入力）
├── プロンプトログ → 何を考えていたか（思考パターン）
├── 日記・断片メモ → 何を感じていたか（感情パターン）
├── 夢・目標     → 何を大事にしているか（価値観）
├── タスク完了   → 何を成し遂げたか（行動パターン）
├── 成果物       → 何を作ったか
└── 意思決定ログ → なぜそう判断したか

    ↓ AIが統合分析

理解モデル（AI内部）
├── 感情の周期パターン（曜日・時間帯・イベントとの相関）
├── 価値観マップ（何に喜び、何に不安を感じるか）
├── 思考の傾向（どんな問いを立てるか、何を重視するか）
├── 行動パターン（いつ何をするか、何が得意か）
├── 夢への距離（どの夢に近づいているか、停滞しているか）
└── コンディション推定（調子が良い/悪い）

    ↓ AIが出力

アクション（出力）
├── Morning Briefing: 今日のフォーカス提案
├── 感情サポート: しんどい時の励まし
├── 夢アラート: 近づいている夢への気づき
├── パターン発見: 「火曜に不安が高まる傾向」
├── 振り返りレポート: 週次/月次のナラティブ
└── 暗黙の最適化: 毎回の指示なしで意図通りに動く
```

## 5. 技術的な検討事項

### Single HTML → どうする？

| 選択肢 | メリット | デメリット |
|---|---|---|
| A: Single HTML 維持 | 配布容易、既存資産活用 | モバイル体験に限界。複雑なUIは困難 |
| B: Next.js + PWA | モバイルファースト、コンポーネント化 | 既存コードの書き直し |
| C: React Native (Expo) | ネイティブ体験、diary と技術統一 | 開発コスト大 |
| **D: 段階的移行** | Phase 1-2はHTML維持、Phase 3でPWA化 | 2回作ることになるリスク |

**推奨**: D案（段階的移行）
- Phase 1-2: 現在のSingle HTMLを改修。Today画面・感情UI・夢リストを追加
- Phase 3: PWA化して本格的なモバイル対応。このタイミングで diary の技術（Next.js）に統合

### AI処理の設計

| 処理 | タイミング | 方式 |
|---|---|---|
| 感情分析 | diary_entries INSERT時 | Supabase Edge Function → Gemini Flash |
| パターン発見 | 週次バッチ | Edge Function (cron) |
| 夢の達成検出 | diary_entries INSERT時 | Edge Function → 夢リストとのマッチング |
| Morning Briefing | ログイン時 | クライアントサイド → AI Chat API |
| 週次ナラティブ | 金曜夕方 | Edge Function (cron) |
