# ハンドオフ検出ルール

> 秘書（/company）が成果物を確認し、次の部署を自動起動するためのルール。

## 検出方法

部署の成果物ファイル（.md）を読んだ際に、以下のパターンを検出する:

### パターン1: 明示的ハンドオフ
```
→ {部署名}部への依頼
→ {部署名}への依頼
### {部署名}部への依頼事項
```

### パターン2: 暗黙的ハンドオフ
成果物の内容から推測:
- 「実装が必要」「コードを書く」 → システム開発部
- 「資料にまとめる」「プレゼンを作る」 → 資料制作部
- 「チケット化」「タスクに分割」 → PM部
- 「テストが必要」「QA確認」 → QA

## 秘書のアクション

1. 成果物を読む
2. ハンドオフパターンを検出
3. 社長に確認（checkpoint モードの場合）:
   ```
   UXデザイン部の分析が完了しました。
   以下のハンドオフが検出されました:
     → PM部: 改善案をチケット化（3件）
     → AI開発部: APIの仕様策定
   進めてよいですか？
   ```
4. 承認を得たら Agent で該当部署を起動
5. activity_log に記録:
   ```
   action: 'dept_handoff'
   metadata: {
     from: 'ux-design',
     to: 'pm',
     trigger: 'explicit_handoff',
     items: ['改善案のチケット化', '期限設定']
   }
   ```

## 自動起動しない場合

- full-auto モード以外は必ず社長に確認
- 見積もり費用が発生する場合（外部API呼び出し等）
- セキュリティ部の監査結果に問題がある場合
