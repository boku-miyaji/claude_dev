# デモシステム改善設計書

## 改善対象: 4画面

### 1. 面談記録 (meeting-list.tsx)
**問題**: v3で面談2回のデータがあるのに表示されない場合がある
**原因**: mock-data.tsでmeetingNotesはv3のみ。v1/v2ではmeetingNotesが空配列のため、v1→v2→v3のデモフローでは最初は表示されない。v3に切り替えた際にも、storeのcustomerIdフィルタで正しく取得される。問題は面談タブ自体にあるのではなく、v3の面談記録の内容が貧弱（confirmedFactsなどが短い）な可能性。
**対応**: v3の面談記録データを充実させ、面談内容が十分に表示されるようにする

### 2. アジェンダ詳細の経緯セクション (agenda-detail-modal.tsx)
**問題**: 経緯セクション（背景・経緯）はコード上存在するが、面談やアジェンダの作成・更新の経緯が十分に表示されていない
**対応**: 既存の「背景・経緯」タイムラインを強化。面談でどんな情報が得られたか、アジェンダがどう変化したかを時系列で追えるようにする。VersionLogにsummaryが空のエントリがある場合に表示が空になる問題を修正

### 3. 提案ストーリー (proposal-story-view.tsx)
**問題1**: 想定ロードマップの短期/中期/長期の時間軸表示が不要 → 削除
**問題2**: 仮説・目的・前提セクションがない
**対応**:
- 想定ロードマップセクションのtimeframeバッジを削除
- S1とS2の間に新セクション「提案の仮説・目的」を追加
  - 仮説: 各アジェンダのwantToAchieve + painPointから構成
  - 目的: この提案で何を達成したいか
  - 前提: concerns[] から構成

### 4. キーパーソンマップ (keyperson-map.tsx)
**問題**: 子会社へのアプローチ、各人物へのアプローチ戦略、ヒアリング内容が見えない
**対応**:
- KP詳細ダイアログに「アプローチ戦略」セクションを追加（設計原則の「事実のみ」を拡張）
- 面談記録からKP関連のヒアリング内容を自動抽出して表示
- 子会社タブにもアプローチ方針を表示

## 次のステップへの申し送り
- 実装チームは上記4点を順に修正すること
- proposal-story-view.tsxの変更が最も大きい（新セクション追加）
- keyperson-map.tsxはKP詳細ダイアログへの追加が中心
