## 第3部: RFIから読み取れる重要な論点と盲点

### 既存サマリーで不足している視点

1. **Viewerライセンス数の根拠が曖昧**: PwC=1,000名、Salesforce=3,452〜3,740名と大幅に異なる。SOMPOケアの実利用者数（本部/施設長/現場/経営層）の正確な把握が不可欠
2. **ETL層の重要性が過小評価されている**: 現行Foundryの最大価値はETL/パイプライン部分。BI選定以上にETL移行の工数とリスクが大きい
3. **SOMPOグループ横断データ基盤との整合**: 2026/10開始予定の「グループ顧客共通基盤」との技術的整合が未検討
4. **Foundry内PySparkコードの資産価値**: 富士通のみが移行ノウハウを持つ。他社にとってはブラックボックス → 移行PoC必須
5. **介護業務システムリプレース（2028/5）との連携**: 新分析環境のデータソースが根本的に変わる可能性がある
6. **データガバナンス/カタログの位置づけ**: PwCのみMDM、デロイトのみUnity Catalog/Data Catalog、他社は軽視気味
7. **非構造化データ（ケア記録テキスト・音声等）への対応**: AI/ML拡張の核心だが、具体的な方針がRFIレベルでは不足
